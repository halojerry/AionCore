## 概览
本文档定义了互动教学通关游戏的 game_config.json 数据结构。脚本 `build_game.py` 读取此 JSON，将图片转为 Base64 嵌入，生成单文件 HTML 通关游戏。

## 核心内容

### 1. JSON 完整结构

```json
{
  "title": "{课题} · 通关游戏",
  "intro": "游戏背景介绍，1-2 句话说明游戏目标",
  "cover_image": "images/cover.jpg",
  "start_scene_id": "1",
  "scenes": [
    {
      "id": "1",
      "text": "场景描述（1-2句话，20-50字）",
      "image": "images/scene1.jpg",
      "image_prompt": "用于生成配图的提示词",
      "choices": [
        {
          "text": "选项A（正确选项）",
          "next_scene_id": "2",
          "is_correct": true,
          "feedback": "选择正确的鼓励语"
        },
        {
          "text": "选项B（干扰项）",
          "next_scene_id": "1",
          "is_correct": false,
          "feedback": "为什么不对，以及提示"
        },
        {
          "text": "选项C（干扰项）",
          "next_scene_id": "1",
          "is_correct": false,
          "feedback": "为什么不对，以及提示"
        }
      ]
    }
  ],
  "review": {
    "knowledge_points": ["核心知识点1", "知识点2", "知识点3"],
    "discussion": ["课堂讨论题1", "讨论题2"]
  }
}
```

### 2. 各科目游戏设计逻辑

#### 2.1 语文 — 故事通关
- **视角**：以第一人称或旁观者视角，代入课文故事
- **场景拆分**：按课文情节发展拆成 5-10 关
- **选项内容**：围绕情节理解和人物行为选择
- **示例**：《小蝌蚪找妈妈》— "帮助小蝌蚪找妈妈"视角，每关选择正确的问路对象或识别特征

#### 2.2 数学 — 算术闯关
- **视角**：以"数学探险"或"挑战关卡"为框架
- **场景拆分**：知识点拆成 5-10 个由易到难的题目
- **选项内容**：每关一道计算或推理题，3 个选项含 1 个正确答案
- **示例**：《分数加减法》— "分数王国的秘密"框架，每关一道分数运算题

#### 2.3 英语 — 情景对话
- **视角**：以日常交际场景为背景
- **场景拆分**：对话拆成 5-10 个交际场景
- **选项内容**：每个场景选正确回应推进对话
- **示例**：《At the Restaurant》— 点餐场景，选正确英文回应服务员

#### 2.4 科学 — 实验探索
- **视角**：以"小小科学家探索"为框架
- **场景拆分**：实验步骤拆成 5-10 个关卡
- **选项内容**：选正确实验步骤或科学解释推进
- **示例**：《种子发芽实验》— 设计种子发芽实验的步骤选择

#### 2.5 道法 — 道德抉择
- **视角**：以生活情境为背景
- **场景拆分**：设置 5-10 个道德/法律情境
- **选项内容**：选正确行为推进剧情
- **示例**：《诚实守信》— 面对各种情境，选择诚实的行为

### 3. 场景拆分原则

1. **数量**：5-10 个场景（默认 7 个）
2. **长度**：每个场景描述 1-2 句话，20-50 字
3. **逻辑连贯**：场景之间要有逻辑递进关系
4. **知识点覆盖**：确保关键知识点都体现在场景中
5. **难度递进**：由易到难排列
6. **最后一个场景**：设计为"通关"场景，有庆祝感

### 4. 选项设计规范

#### 4.1 基本要求
- 每个场景**恰好 3 个选项**
- **恰好 1 个正确**选项（is_correct: true）
- **恰好 2 个干扰**选项（is_correct: false）
- 每个选项文字精炼为一句话，**最多 30 字**

#### 4.2 正确选项设计
- 严格遵照课文/知识原意
- 选项内容明确、不含歧义
- feedback 给出正面鼓励："回答正确！+ 简短的解析"

#### 4.3 干扰选项设计
- 与正确答案相似但有细微差别
- 符合学生常见的错误认知或误解
- 不能明显离谱或搞笑
- feedback 给出引导："再想想 + 提示方向"

#### 4.4 选项反馈（feedback）规则
- 正确选项 feedback：肯定 + 简短知识点解析
- 错误选项 feedback：温和否定 + 提示方向
- **选错后留在当前场景重新选择**（next_scene_id 指向当前场景 id）
- **选对后进入下一场景**（next_scene_id 指向下一个场景 id）

### 5. 配图要求

#### 5.1 图片类型和数量
- 封面图 ×1：游戏开始画面，展现游戏主题
- 场景图 ×N：每个场景配一张图（5-10 张）

#### 5.2 风格要求
按科目使用不同配图风格（详见 SKILL.md 科目速查表）

#### 5.3 图片路径
- JSON 中使用相对路径：`"image": "images/scene1.jpg"`
- 图片实际保存在 `user-data/images/` 目录
- 脚本会将图片转为 Base64 嵌入，最终为单文件 HTML

#### 5.4 image_prompt 字段
每个场景提供 `image_prompt` 字段，描述配图内容，供图像生成使用：
- 语文：卡通/水墨插画提示词
- 数学：几何示意图提示词
- 英语：卡通场景提示词
- 科学：实物/示意图提示词
- 道法：生活场景提示词

## 注意事项

- JSON 文本字段中的引号必须使用单引号（'）
- 图片在调用脚本前必须先生成
- 选错场景的 next_scene_id 必须等于当前场景 id（原地重选）
- 选对场景的 next_scene_id 指向下一个场景 id
- 最后一个场景的正确答案可指向自身或特殊值表示通关
- review 字段在通关后展示，用于课堂讨论
