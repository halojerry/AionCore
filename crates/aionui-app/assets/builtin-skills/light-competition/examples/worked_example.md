# Worked Example：基层眼底病智能筛查系统（大创 + 创业组 双形态走查）

> 这是一条假想但自洽的主线，展示 light-competition 的完整流程。所有数字在六个产物间一致，
> 演示评委最爱追问的"BP 说 100 万用户、财务却按 10 万算"这类自相矛盾如何被避免。

**项目一句话**：为县域基层医疗机构提供手机端 AI 眼底病分级筛查，把早期糖网漏诊率降下来。

**关键参数（贯穿全程的单一真相源，改一处需同步全部）**：

| 参数 | 取值 | 出现在哪些材料 |
|---|---|---|
| 项目周期 | 2026-09 ~ 2028-08（约 20 个月即结题）| 申报书 §9 / roadmap JSON / 预算时间线 |
| SOM（3 年可获取） | 6.4 亿元/年 | 核心页⑧ / BP §5 / market JSON / 财务预测 |
| 现实渗透率 | 县域机构 8% | BP §5 假设登记表 / 财务预测 |
| 验证集灵敏度门槛 | ≥90% | 核心页④ / roadmap go/no-go / 答辩 QA |
| 总预算 | 8 万元（大创校级）| 申报书 §11 / budget_table |

---

## Step 1 · 先定位赛事 + 产 rules_checklist

<!-- 同一个项目可投两个方向，评审重点完全不同（anti_patterns E1）。 -->

| 形态 | 赛事/组别 | 评审重点 | 用哪套模板 |
|---|---|---|---|
| 学术向 | 大创·创新训练 | 学生主体、培养价值、研究深度 | `templates/application_draft.md` |
| 创业向 | 中国国际大学生创新大赛·本科创业组 | UVP、壁垒、市场测算、社会价值 | `templates/business_plan.md` |

动笔前先按 SKILL「先定位」步产出 `material_checklist.md`，勾选当届组别/页数/重点领域/提交物/AI 规定已对齐（见 references/competition_rules_cn.md）。本例假设已确认 2026 当届规则。

---

## Step 2 · 写核心页（八段，借 NIH Specific Aims，最先打磨）

<!-- 完整八段写法见 assets/aims_zh_guide.md。这里是填好的成品。 -->

> **①Hook**：我国糖尿病患者超 1.4 亿，其中约 1/3 会发展为糖尿病视网膜病变（糖网），而县域基层眼科医生缺口巨大，早期糖网在可逆阶段的漏诊率居高不下。
> **②Gap**：现有 AI 筛查模型依赖高质量台式眼底相机，在基层手机成像条件下准确率骤降，难以下沉到最需要的县域。
> **③长期目标**：让眼底慢病早筛在基层"人人可及"。
> **④本项目目标**：20 个月内做出手机端轻量分级模型（验证集灵敏度≥90%）+ 县医院前瞻验证。
> **⑤中心假设**：在低质手机眼底图上，通过质量自适应 + 轻量化网络可达到临床可用的分级灵敏度。
> **⑥立论依据 + 前期基础**：团队已用公开数据集预训练出基线模型（灵敏度 87%），证明路线可行。
> **⑦子目标**：见申报书 §4 三个子目标。
> **⑧Payoff**：产出可下沉县域的筛查工具，降低早期漏诊，预期 3 年可获取市场（SOM）6.4 亿元/年。

注意 ④的"≥90%"、⑧的"6.4 亿元"就是关键参数表里的值，后面每份材料都复用同一数字。

---

## Step 3 · 申报书 §4 三个子目标（彼此独立但互补，各配备选方案）

<!-- 关键：子目标互不依赖，避免"目标2/3建立在目标1必须成功上"（anti_patterns C4）。 -->

| 子目标 | 理由 | 工作假设 | 方法概要 | 预期产出 | 备选方案 |
|---|---|---|---|---|---|
| 1 质量自适应预处理 | 手机图质量参差 | 质量分级可前置过滤 | 图像质量评估网络 | 质控模块 | 退化为人工质量门 |
| 2 轻量化分级模型 | 端侧算力受限 | <5M 参数可达临床灵敏度 | 知识蒸馏 + 量化 | 模型 v1（灵敏度≥90%）| 退到云端推理 |
| 3 县医院前瞻验证 | 实验室≠真实场景 | 现场灵敏度可达标 | 试点协议 + 前瞻采集 | 验证报告 | 缩小到单中心 |

三个目标可并行推进，任一不达标不拖垮全盘——这正是评委判断"项目风险是否集中"的关键。

---

## Step 4 · 进度安排 → roadmap_gen.py（申报书 §9 的可视化）

把申报书 §9 里程碑表整理成 JSON（结构见 `scripts/roadmap_gen.py --emit-sample`），喂脚本出甘特图 + 技术路线图：

```bash
# 导样例（本例 JSON 即此样例，已是基层眼底筛查主线）
python scripts/roadmap_gen.py --emit-sample roadmap.json
# 出甘特图 + 技术路线图（短周期竞赛可加 --granularity week）
python scripts/roadmap_gen.py roadmap.json --kind both --out figures/
```

JSON 里 `start: "2026-09"` 与关键参数表的项目周期一致；`go/no-go: 验证集灵敏度≥90%` 与核心页④、答辩 QA 同一门槛。**进度图是可再生产物，材料定稿后可删，留 JSON 即可重生成。**

---

## Step 5 · 市场分析 → market_charts.py（BP §5 的四图）

创业向才需要。把 BP §5 的 TAM/SAM/SOM + 竞品 + 五力 + 风险整理成 JSON（结构见 `scripts/market_charts.py --emit-sample`）：

```bash
python scripts/market_charts.py --emit-sample market.json
python scripts/market_charts.py market.json --kind all --out figures/
```

**数据一致性是这一步的命门**：JSON 里 `SOM.value: 6.4`（亿元）= 核心页⑧ = 关键参数表 = 财务预测的收入天花板。`SOM.note` 明确写"SOM→财务预测用户数同源"，杜绝"BP 说 6.4 亿、财务按 1 亿算"被评委一问就崩（anti_patterns C3/C5）。脚本强制 TAM(320)≥SAM(64)≥SOM(6.4)，层级倒挂会直接报错。

---

## Step 6 · 经费预算 → budget_table.md（申报书 §11）

套 `databases/db08-ip-materials/budget_template.md` 第 2 节科研支出预算表。大创校级总额按关键参数表 8 万元，逐项对齐研究活动（anti_patterns D4：不堆在某一类）：

| 预算科目 | 金额（元） | 对应研究活动 | 测算依据 |
|---|---|---|---|
| 数据采集/标注费 | 30000 | 子目标 1/3：≥3000 张眼底图标注 | 10 元/张 × 3000 |
| 测试/计算费 | 20000 | 子目标 2：模型训练 GPU 租用 | 云 GPU 4 元/时 × 5000 时 |
| 差旅/调研费 | 15000 | 子目标 3：县医院前瞻验证现场 | 2 人 × 3 次 × 2500 |
| 文献/版面费 | 10000 | 结题论文发表 | 1 篇版面费 |
| 耗材/其他 | 5000 | 原型设备、打印 | — |
| **合计** | **80000** | — | 与 §9 五个研究阶段一一对应 |

预算↔活动↔时间线↔人员内部一致：差旅费对应 §9 的"县医院前瞻验证"阶段（2027-07 起），不会出现"某阶段没活动却报差旅"的脱节。

---

## Step 7 · 交付前反模式自查（references/anti_patterns.md 一页速查表）

逐条扫一遍，命中即改：

| 类 | 本例自查结果 |
|----|-----------|
| 概念 | ✅ 痛点有数字（1.4 亿患者）；✅ 创新说清"为什么难"（手机低质图）；✅ 壁垒是质量自适应算法+县域数据非"先发" |
| 写作 | ✅ 摘要有模型名+灵敏度数字；✅ 进度每步有交付物+go/no-go；✅ 社会价值量化（漏诊率/SOM） |
| 技术 | ✅ 有前瞻验证（相当于灵敏度检验）；✅ 市场自下而上（8% 渗透率×ARPU）；✅ 子目标互不依赖 |
| 格式 | ✅ 组别/重点领域已对齐当届（material_checklist）；✅ 无占位符残留 |
| 策略 | ✅ 已确认赛事/组别；✅ 卖点排序（先讲灵敏度+下沉）；✅ 体现学生主体（大创形态） |

最后做评审模拟，预演答辩 QA（`templates/defense_qa.md`）：评委最可能问"手机图灵敏度真能到 90%？"→ 答辩备份附录放灵敏度分析表。

## 产物落盘清单（CONVENTIONS §6.1）

走完全程，`competition/` 目录下应有：

```
competition/
├── application_draft.md     # 学术向（大创）→ 套 templates/application_draft.md
├── business_plan.md         # 创业向 → 套 templates/business_plan.md
├── abstract.md              # 项目摘要 → 套 templates/abstract.md
├── defense_qa.md            # 答辩预演 → 套 templates/defense_qa.md
├── budget_table.md          # 预算表（Step 6）
├── material_checklist.md    # 当届规则确认 + 交付清单（Step 1 起就建）
├── roadmap.json             # 进度（可再生图的源，图本身用完即删）
└── market.json              # 市场（创业向；同上）
```

**一致性收尾（a07）**：关键参数表里每个数字，在上面所有材料中只有一个值。这是评委追问时项目"不自相矛盾"的根本保证，也是 light-consistency(a07) 跨材料核查的对象。
