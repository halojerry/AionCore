# 跨赛事反模式手册（anti_patterns.md）

> 服务 light-competition（m17）。汇总数模/统计建模/互联网+/挑战杯/大创/科研训练的**常见失分与出局点**。
> 分五类：**概念 / 写作 / 技术 / 格式 / 策略**。每条 = 症状（怎么看出来）→ 为什么被扣分（评委逻辑）→ 怎么改（可执行）。
> ⚠ 诚实声明：各赛事确切评分表逐年逐赛区调整且多数不完全公开；下文"被扣分原因"是公开赛制长期通行口径下的**评审逻辑**，非引用某届确切分值。写前以当届评审规则为准。

---

## A 类 · 概念反模式（想清楚没有）

### A1 把"领域热"当成"问题真"
- **症状**：开篇"随着人工智能/大数据/双碳的快速发展……"，但说不出**谁、在什么场景、损失多少**。
- **为什么被扣**：评委见过上百个这种开头，无法判断问题是否真实存在、值不值得做；显得是为了套热点硬凑。
- **怎么改**：开场钩子用**具体数字**点出痛点规模（人群/比例/成本），痛点要有一手调研或可核查数据。见 `assets/aims_zh_guide.md` 第 1 段。

### A2 创新点是"首次把 A 用到 B"却不说为什么难
- **症状**：创新点写成"首次将 Transformer 用于 XX""国内首个 XX 平台"。
- **为什么被扣**：套用现成方法到新场景若无技术障碍，只是工程拼装，不构成创新；评委会问"那别人为什么没做"。
- **怎么改**：按五维写创新（概念/方法/整合/转化/规模），明确**现有方案的局限**以及你**如何克服**。可量化、可展示（复用 m03/m04 判断）。

### A3 把"先发/团队/努力/资源"当壁垒（Unfair Advantage）
- **症状**：商业计划书壁垒一栏写"我们团队执行力强""起步早""有学校支持"。
- **为什么被扣**：这些都可被复制或超越，不是难复制壁垒；双创评委专挑这点追问。
- **怎么改**：壁垒写**真正难复制**的：专利/独家数据/算法/网络效应/独家渠道或资质。Lean Canvas 里 Unfair Advantage 最难填也最关键。

### A4 逻辑不闭环 / 纯 PPT 概念
- **症状**：有宏大愿景，但没有 MVP、原型、或任何已验证环节；问题→方案→变现链条断裂。
- **为什么被扣**：评委容忍"还没盈利"，但不容忍"逻辑不通/无任何证据"。
- **怎么改**：哪怕创意组，也要给最小可行产品或原型证据；用 Lean Canvas 九格先自检闭环再展开。

## B 类 · 写作反模式（表达没到位）

### B1 摘要写成"问题重述"（数模重灾区）
- **症状**：CUMCM/MCM 摘要大段复述题目，或从引言复制粘贴，最后才草草提结论。
- **为什么被扣**：摘要是决定初筛/拉开差距的部分（CUMCM 决定初筛；MCM/ICM 给极高权重，官方明确告诫别写成问题重述）；评委先看摘要，写不好直接出局。
- **怎么改**：摘要**最后写**；写清**模型名称 + 关键方法 + 最重要结论数字**；每问有"模型→求解→检验"闭环。

### B2 笼统的进度安排"第一阶段调研"
- **症状**：进度表写"第一阶段：文献调研；第二阶段：实验；第三阶段：总结"。
- **为什么被扣**：无法验证、无交付物，评委看不出可行性，像凑数。
- **怎么改**：借 writing-plans 思路，把阶段拆成**可验证小步**，每步有**交付物**与里程碑/go-no-go 节点（见 `scripts/roadmap_gen.py` 生成路线图/甘特图）。

### B3 价值段喊口号无数据
- **症状**："本项目具有重大社会价值，将助力乡村振兴/推动行业进步"。
- **为什么被扣**：双创赛事明确考察社会价值（带动就业等），空话无法评估。
- **怎么改**：价值量化——带动多少就业、降低多少成本、覆盖多少人群，给测算依据。

### B4 自卖自夸形容词堆砌
- **症状**："国际领先""颠覆性""完美解决"等无证据的最高级。
- **为什么被扣**：评委要证据不要形容词；过度自夸反而降低可信度。
- **怎么改**：show don't tell——用对比数据、用户证言、实测指标说话。

## C 类 · 技术反模式（专业硬伤）

### C1 缺灵敏度分析与误差检验（数模重灾区）
- **症状**：模型建完直接报结果，无灵敏度分析、无结果检验、无稳健性讨论。
- **为什么被扣**：灵敏度/误差分析是公认拉分项，缺失说明对模型可信度没把握。
- **怎么改**：每个模型配灵敏度分析（关键参数扰动看结果变化）+ 结果检验 + 优缺点讨论。

### C2 算法炫技压过真实问题（统计建模重灾区）
- **症状**：堆叠各种花哨模型，但数据来源存疑、变量无解释、推断不严谨。
- **为什么被扣**：统计建模大赛重**数据真实、变量解释、统计推断严谨、现实意义**，而非算法堆砌。
- **怎么改**：数据标明真实来源；多模型对比并解释优劣；统计推断给置信区间/检验；落到现实意义。（统计检验直接复用 m06 result-analysis 的共享资产 `code_assets/stats_tests.py`：Welch t 检验 / BH-FDR 多重比较校正 / Wilson 比例置信区间 / bootstrap CI，不要重造轮子。）

### C3 市场规模自上而下虚高
- **症状**："市场 1 万亿，我们只要占 1% 就是 100 亿"。
- **为什么被扣**：典型自上而下虚高，假设经不起追问；双创评委一问就崩。
- **怎么改**：TAM→SAM→SOM **自下而上**测算（人口基数×转化过滤×现实 ARPU），每层假设进**假设登记表**让评委直接质疑输入，再用竞品营收/公开财报**交叉校验**；数据≤2 年标出处。

### C4 子目标互相依赖，一处崩全盘崩
- **症状**：研究内容里目标 2、3 都建立在"目标 1 必须成功"之上。
- **为什么被扣**：风险集中，评委会判定整个项目高风险。
- **怎么改**：子目标设计成**彼此独立但互补**；每条配**备选方案（contingency）**应对失败。

### C5 财务只有收入没有成本和现金流
- **症状**：财务预测只列收入曲线一路上扬，无成本结构、无现金流转正点。
- **为什么被扣**：不现实，看不出何时能自我造血，创业可行性存疑。
- **怎么改**：3 年损益 + 现金流，标出**现金流转正时点**；成本结构对齐 Lean Canvas。模板见 db08/budget_template.md。

## D 类 · 格式反模式（低级失分）

### D1 超页 / 把附录代码当"页外内容"（MCM/ICM 重灾区）
- **症状**：以为附录、代码、参考文献不计页数。
- **为什么被扣**：MCM/ICM **25 页硬上限**，包含 Summary Sheet+正文+参考文献+目录+附录+代码全部计入，**唯一例外是"Report on Use of AI"**（必填、不计页）。超页直接出问题。
- **怎么改**：提交前数总页数（含附录代码）；压缩代码展示，长代码截关键段。AI 使用说明如实写、单列。

### D2 套用旧届模板 / 组别名称过时
- **症状**：用上一届的组别名（如"初创组/成长组"）、旧重点领域、旧评分项。
- **为什么被扣**：组别名称和重点领域**逐年改**（如互联网+→中国国际大学生创新大赛，组别已更新），套旧模板显得不用心。
- **怎么改**：写前下**当届官方评审规则压缩包**对齐组别、重点领域、提交物清单（db08 标注规则逐年改）。

### D3 占位符/未替换文字残留
- **症状**：交付材料里残留"xxx 公司""lorem ipsum""【此处填写】""某某"。
- **为什么被扣**：低级失误，直接拉低专业印象。
- **怎么改**：交前 grep 查残留占位（`xxxx|lorem|【|某某|待填`）；PPT 走视觉 QA 闭环（转 PDF→逐页转图→新眼睛检查）。

### D4 预算分类不合理 / 与研究内容脱节
- **症状**：经费堆在某一类（如大额设备费），或预算项与研究活动对不上。
- **为什么被扣**：大创等明确要求预算分类合理、与研究内容匹配，不可堆砌。
- **怎么改**：预算逐项对齐研究活动（材料费/测试费/差旅/版面费等）；保证**预算↔活动↔时间线↔人员**内部一致。

## E 类 · 策略反模式（赛道与定位）

### E1 没确认是哪个赛事/哪个杯/哪个组就动笔
- **症状**：把"大挑"(学术)当"小挑"(创业)写，或创意组按创业组要求堆财务。
- **为什么被扣**：评审重点完全不同（大挑偏学术深度，小挑偏商业可行），错位等于答非所问。
- **怎么改**：动笔前先定位赛事/项目/组别，对齐当届官方规则的评审重点（见 SKILL.md "先定位"）。

### E2 卖点平铺直叙，不分主次
- **症状**：把所有功能/亮点平均用力罗列，评委记不住核心。
- **为什么被扣**：评委注意力有限，没有记忆点；显得团队自己没想清楚最强的是什么。
- **怎么改**：借 content-strategy **加权打分**（客户影响 40%+契合 30%+潜力 20%+资源 10%）给卖点排序，路演先讲最强的一两个。

### E3 忽视"学生主体/培养价值"（大创/科研训练重灾区）
- **症状**：大创申报书写得像导师的横向课题，看不出学生做了什么、成长了什么。
- **为什么被扣**：大创重**学生主体与过程性成长**，不只看结果。
- **怎么改**：突出学生在各环节的主导角色、分工、能力提升；进度安排体现培养路径。

### E4 不做评审模拟就交稿
- **症状**：自我感觉良好直接提交，没人扮评委挑刺。
- **为什么被扣**：能被现场问倒的漏洞，提前都能补；不做模拟等于裸奔。
- **怎么改**：出稿后用评委视角自审（类比 m14）+ devil's-advocate 对抗轮：基金类过 NIH 五项/NSF 两项；数模回看摘要与灵敏度；创业类回看 UVP/壁垒/市场测算是否经得起追问。预演答辩 QA。

## 一页速查（交稿前最后扫一遍）

| 类 | 一句话自查 |
|----|-----------|
| 概念 | 痛点有数据吗？创新说清"为什么难"了吗？壁垒真难复制吗？逻辑闭环吗？ |
| 写作 | 摘要写成重述了吗？进度有交付物吗？价值量化了吗？ |
| 技术 | 有灵敏度/检验吗？数据真实标源吗？市场自下而上吗？子目标互不依赖吗？财务有现金流吗？ |
| 格式 | 超页吗（含附录代码）？用旧届模板了吗？有占位符残留吗？预算对齐活动吗？ |
| 策略 | 确认赛事/组别了吗？卖点排序了吗？体现学生主体了吗？做评审模拟了吗？ |
