# design_language_extract — 视觉设计语言抽取模板（空白）

> **资产放置原则（SKILL 第22行）**：本文件是 `assets/` 下的**空白模板**；
> 真实抽取产物落 `databases/db09-projects/projects/<项目>/`，DTCG 副本与
> `databases/db05-frontend-styles/design_tokens.template.json` 同源。
>
> **诚实声明**：本抽取为**人工/半人工**流程。从截图取色受 JPEG 压缩、抗锯齿、
> 屏幕色彩管理影响会偏差；抽出的"隐含规则"可能并非原设计意图。
> 凡标 ⚠ 的栏目**必须人工确认并签字**后方可写入 db05/db09 视觉 SSOT。
> extract-design-system 不自动产出可信 token，consistency_audit.py 也不核视觉 token。

## 3. ⚠ 隐含规则需人工确认（核对栏）

逐条勾选并签字，未签字不得作为 SSOT：

- [ ] 取色已校正压缩/抗锯齿偏差（对比原始矢量/CSS 源而非截图）——确认人：____
- [ ] 语义色映射符合原设计意图（不是"我猜这是危险色"）——确认人：____
- [ ] 字号阶是有意的 modular scale 而非偶然值——确认人：____
- [ ] 间距确属 4/8pt 栅格而非肉眼凑整——确认人：____

**形态 B — DTCG json（可选，机读增强，仅严格场景）**：把规范卡填进

仅当确需 CSS/SCSS/JS 多平台同步时再引入。
