# 文件理解笔记 · {{file_name}}

> 一句话定位：这份文件是什么、为什么要读它、读完要支撑哪个下游动作。

## 0 · 文件元信息

| 字段 | 内容 |
|------|------|
| **文件名/路径** | `{{path}}` |
| **格式** | PDF / DOCX / PPTX / XLSX / CSV / 图片 / 视频 / 代码 / 压缩包 / 其他 |
| **读取方式** | 宿主原生 Read / pdf_ops / docx_read / xlsx_read / pandoc / markitdown / OCR / ffmpeg+whisper |
| **页数/规模** | 页数 / 字数 / 行列 / 时长 |
| **作者/来源** | 作者、机构、日期（取自核心属性/元数据，未知写未知） |
| **版权/敏感** | 是否受版权全文不外传；是否含密钥/隐私（按 key 名引用，不回显值） |
| **可信度** | 一手/二手、是否权威、是否最新 |

## 1 · 结构与逻辑（骨架）

- **章节/段落骨架**：列大纲层级（用 `docx_read.py headings` / PDF 目录提取）。
- **论证链 / 叙事骨架**：问题→方法→证据→结论的推进路径，或叙事顺序。
- **章节关系**：哪节支撑哪节、依赖/递进/对比关系。
- **跨文件关系**（批处理时填）：本文件与其他材料的引用/补充/冲突关系。

## 2 · 关键内容（讲了什么）

| 维度 | 内容 |
|------|------|
| **核心问题** | 这份文件要解决/回答什么 |
| **方法/做法** | 用了什么方法、流程、设计 |
| **数据** | 关键数据点、数据集、样本量、统计口径（来自表格/XLSX 画像） |
| **结果** | 主要发现、指标、结论性数字（标注出处页/表） |
| **结论/主张** | 作者的核心 claim 与边界条件 |
| **未决/存疑** | 缺失信息、未验证主张、需进一步核实之处 |

## 3 · 格式与要求（模板/约束）

- **模板规范**：页边距/纸张/字号/行距/编号（来自 `docx_read.py layout` / `read_runs`）。
- **引用风格**：APA / GB-T 7714 / IEEE 等；参考文献格式。
- **字数/篇幅约束**：投稿/任务的硬上限。
- **隐含要求**：从文件推断但未明说的约束（如盲审要求、术语统一、占位符残留）。

## 4 · 视觉风格（供 a05/m16/m11）

- **配色**：主色/辅色、是否成体系（截图取色）。
- **版式**：栏数、留白、对齐、标题层级（PPT 36-44pt / 正文 14-16pt 量级）。
- **图表风格**：图型、标注规范、误差棒/单位/panel 标号习惯。
- **可沉淀去向**：前端风格→db05、PPT 风格→db06、图表风格→db07。

## 5 · 可复用内容（能直接拿来用的）

| 类型 | 位置 | 复用去向 |
|------|------|----------|
| 段落/表述 | 第X节/页 | 论文/报告改写 |
| 数据/表格 | 表X/sheet X | 重绘图表 / 数据分析 |
| 图/示意 | 图X | 复用或重绘 |
| 结构/模板 | 全文骨架 | 套用为新文档模板 |

## 6 · 文件 → 下游技能动作映射（转化为可执行任务）

> 明确"这个文件→接下来能做什么"，每条给出触发条件与目标技能编号。

| 若文件含 | 下游动作 | 目标技能 |
|----------|----------|----------|
| 论文正文/审稿意见 | 论文修改、润色、回应 | m08(润色) / m13(rebuttal) |
| 实验数据/结果表 | 结果分析、显著性检验 | m06(result-analysis) |

| 杂乱项目文件 | 项目结构整理 | a06(project-structure) |
