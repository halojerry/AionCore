# 立项卡 · I-01

| 字段 | 内容 |
|------|------|
| **idea_id** | I-01 |
| **标题** | 自监督时序对比学习用于奶山羊发情早期识别 |
| **一句话机制** | 用时序对比学习从加速度序列自监督学习发情前驱运动模式，把"密集人工标注"降到"少量锚点标注" |
| **新颖性主张** `(m04 复核)` | ②增量：把已知的时序对比学习（如 TS-TCC）系统化扩展到畜牧发情早期预警这一新任务设定，明说是跨域扩展而非全新方法 |

## 最近邻工作（≥3 篇，带检索留痕）`(m04 复核 · 核心撞车一票否决)`

| # | 文献（标题 / 作者年份 / DOI 或 OpenAlex id） | 检索留痕（关键词 / 库 / HTTP / 命中） | 与本 idea 核心主张是否实质等价 |
|---|---|---|---|
| 1 | Goat estrus detection with supervised CNN / Smith 2023 / 10.0000/ex.a | "estrus accelerometer goat"×OpenAlex HTTP200 命中12 | 否——他用全监督 CNN、需密集标注，本 idea 主打自监督降标注 |
| 2 | TS-TCC: time-series contrastive learning / Lee 2022 / 10.0000/ex.b | "time series contrastive learning"×S2 HTTP200 命中30 | 否——通用方法、非畜牧，本 idea 是其跨域扩展（承认借用其框架） |
| 3 | Self-supervised livestock behavior / Wang 2021 / 10.0000/ex.c | "self-supervised livestock behavior"×OpenAlex HTTP200 命中8 | 否——做采食/反刍分类，未涉发情早期预警这一时间敏感任务 |

- **阴性证据**：用 "self-supervised + estrus + early prediction"、"contrastive + goat behavior" 两组关键词 × OpenAlex+S2 两库交叉，均无同时命中"自监督+发情+早期"三要素的工作，缺口真实。
- **撞车自评**：② 前作做过相关但有明确实质扩展 → 点名 Lee 2022（借框架）+ 讲清 delta（见下）。

## 与最近邻的差异 `(m04 复核)`

相对 Smith 2023：不需逐帧密集标注，少量锚点即可（标注成本预期降 ~70%）。相对 Lee 2022：引入畜牧领域先验（昼夜节律 + 个体基线归一化）作为对比学习的正样本构造约束，非直接套用。预期可量化 delta：发情识别**提前量从赛事基准的 ~2h 提升到 ~6h**（最小验证实验验证）。

## 风险与失效条件

- **最可能失败点**：自监督学到的表征对"发情"这一稀有事件不敏感（正样本太少）。
- **失效条件**（量化）：若发情类样本 < 100 段，则对比学习正样本对不足，表征坍塌、提前量无提升。
- **二阶后果**：若失效，退化为需监督微调，标注成本优势丧失。
- **竞争性解释**：提前量提升可能来自个体基线归一化（工程 trick）而非对比学习本身——m04 会查，最小验证实验须做消融分离两者。

## 最小验证实验

数据：自采 dairygoat-sensor 3000 段；baseline：Smith 2023 监督 CNN；指标：发情识别提前量(h) + F1；阈值：提前量 ≥ baseline+3h 且 F1 不低于 baseline-0.02 算机制成立。消融：去掉对比学习预训练（只留归一化）看提前量是否仍提升，分离 trick 贡献。

## 数据可行性 `(m04 复核 · 数据支撑维度)`

- **数据集**：自采 dairygoat-sensor（对齐 db04），加速度计 + 视频帧特征。
- **规模**：3000 段 × 40 只羊；发情类约 400 段（关注稀有类，见 m02 规模预警）。
- **标注方式**：已有兽医发情标注（锚点级）。
- **可得性**：自采，本课题组所有，研究用途无许可问题。

## 算力与成本预估 `(m04 rubric 维度7 · 放行后第一环节算账)`

- **算力**：1× RTX 4090 × ~48h（对比学习预训练 + 微调）。
- **成本**：自有卡，电费量级；无云成本。
- **时间**：约 6 周到可投稿结果。

# 立项卡 · I-02

| 字段 | 内容 |
|------|------|
| **idea_id** | I-02 |
| **标题** | 发情-采食耦合的因果图模型用于混杂解耦的行为识别 |
| **一句话机制** | 把"发情"与"采食活动"的混杂用因果图显式建模，从加速度信号中解耦出发情特异成分，而非端到端黑盒分类 |
| **新颖性主张** `(m04 复核)` | ①新方法：首次把因果解耦（而非相关性分类）引入畜牧行为识别，解决"高活动量同时由发情和采食驱动"的混杂——这是相关性模型无法分离的 |

## 最近邻工作（≥3 篇，带检索留痕）`(m04 复核 · 核心撞车一票否决)`

| # | 文献（标题 / 作者年份 / DOI 或 OpenAlex id） | 检索留痕（关键词 / 库 / HTTP / 命中） | 与本 idea 核心主张是否实质等价 |
|---|---|---|---|
| 1 | Causal representation learning survey / Schölkopf 2021 / 10.0000/ex.d | "causal representation learning"×OpenAlex HTTP200 命中45 | 否——通用理论综述，未涉及行为信号/畜牧 |
| 2 | Activity recognition with HMM / Brown 2019 / 10.0000/ex.e | "goat activity recognition confounding"×S2 HTTP200 命中6 | 否——相关性分类，未做混杂解耦 |
| 3 | Estrus vs feeding behavior overlap / Garcia 2020 / 10.0000/ex.f | "estrus feeding activity overlap dairy"×OpenAlex HTTP200 命中4 | 否——指出了混杂问题但用阈值规则处理，无因果建模 |

- **阴性证据**：用 "causal + estrus + behavior"、"confounding + livestock activity recognition" 两组 × OpenAlex+S2，无任何把因果解耦用于发情行为识别的工作；Garcia 2020 提出了混杂问题正是本 idea 的动机来源但未用因果方法。
- **撞车自评**：③ 无命中且阴性证据充分 → 正常（核心主张"因果解耦做发情识别"无前作）。

## 与最近邻的差异 `(m04 复核)`

相对 Schölkopf 2021：把抽象因果表征学习落到具体的发情-采食混杂场景并给可识别性假设。相对 Brown 2019/Garcia 2020：从"相关性分类/阈值规则"升级到"因果图解耦"，可量化 delta：在采食高峰期的发情识别 F1 预期从 ~0.55（相关性模型受混杂拖累）提升到 ~0.75。

## 风险与失效条件

- **最可能失败点**：因果图的可识别性假设（发情与采食的独立机制）在真实数据上不成立。
- **失效条件**（量化）：若发情与采食的混杂程度（互信息）> 阈值，则解耦不可识别，退化为普通分类。
- **二阶后果**：不可识别则因果建模无增益，徒增复杂度。
- **竞争性解释**：F1 提升可能来自更多参数而非因果结构——须与同参数量的相关性模型对比消融。

## 最小验证实验

数据：dairygoat-sensor + 采食时段标注；baseline：同参数量的相关性分类器；指标：采食高峰期发情 F1；阈值：高峰期 F1 ≥ baseline+0.15 且整体不降算因果解耦有效。消融：打乱因果图结构看增益是否消失。

## 数据可行性 `(m04 复核 · 数据支撑维度)`

- **数据集**：dairygoat-sensor（对齐 db04）+ 需补采食时段标注。
- **规模**：3000 段；需新增采食时段标注 ~3000 段。
- **标注方式**：采食时段需自建标注（回 m02 走标注规范）。
- **可得性**：自采可得；采食标注需额外人工，约 2 周。

## 算力与成本预估 `(m04 rubric 维度7 · 放行后第一环节算账)`

- **算力**：1× RTX 4090 × ~24h（因果图模型比对比学习轻）。
- **成本**：自有卡 + 采食标注人工成本（按 db08 粗算）。
- **时间**：约 8 周（含采食标注 2 周）。
