# db08 — 竞赛申报优秀案例结构骨架（case_skeletons.md）

> 服务 m17(竞赛申报，技能 light-competition)。本文件提供**各赛事高分材料的结构骨架 + 每段要回答什么 + 评审视角**。
> ⚠ 合规：这里只总结**公开可见的结构规律与评审维度**，不缓存任何受版权的官方模板原文或他人作品全文，不提供可直接抄袭套用的成品。具体模板以各校/各赛事**当届官方通知**为准。
> ⚠ 评分权重诚实声明：各赛事**官方评分表的精确分值/百分比逐年逐赛区调整，且多数不完全公开**。下文凡给出维度名称的均为公开赛制长期通行口径；**未给出确切百分比者即因官方未公开或本环境无法当场核实，标【权重待核查】**，写材料时以当届评审规则压缩包为准，不臆造分值。`source_pointer: 各赛事当届《评审规则》压缩包(官网下载) · volatility:high · 不缓存任何分值快照`。

## 1. 互联网+创业组 / 小挑 — 商业计划书 + 路演 PPT

**商业计划书骨架**（评委按"问题真不真→方案行不行→市场大不大→团队靠不靠→财务通不通"读）：
1. 项目概述 / 一句话定位（解决谁的什么痛点）
2. 痛点与市场（Problem + Customer Segments；痛点要有调研证据，非臆想）
3. 产品/解决方案（UVP 独特价值主张；与现有方案差异化、可量化）
4. 技术/壁垒（Unfair Advantage：专利/数据/算法/独家资源，非"先发/团队/努力"）
5. 市场分析（TAM→SAM→SOM 自下而上 + 假设登记表 + 竞品四透镜；数据≤2年标出处）
6. 商业模式（收入来源、定价、渠道、成本结构 = Lean Canvas 展开）
7. 财务预测（3 年损益 + 现金流转正点，见 [budget_template.md](budget_template.md) 第 4 节）
8. 运营与发展规划（里程碑、go/no-go）
9. 团队（分工与能力匹配、与项目强相关）
10. 风险与对策（技术/市场/政策/资金，每条配应对）
11. 社会价值 / 带动就业（双创赛事重点考察项）

**评审维度（公开口径）**：创新性、团队、商业价值 / 成长性、社会价值（带动就业、乡村振兴等导向）、可行性、引领性、（部分赛道）国际化。【各维度确切权重待核查当届评审规则】

**高分特征**：痛点有一手调研；UVP 一句话能记住；市场测算自下而上且交叉校验；财务有现金流转正时点；团队分工与项目强绑定；风险段诚实且有对策。
**常见出局点**：把"先发优势/团队努力"当壁垒；市场自上而下"占 1% 就 X 亿"；财务只有收入没有成本与现金流；社会价值喊口号无数据。

### 1.1 互联网+ 路演 Deck（5–8 分钟）

把上面 11 段 BP 压成 10–12 页路演 Deck（标题用 assertion-evidence 一句断言、页序按 content-strategy 加权、出稿交 m16 渲染+视觉 QA）：

**① BP→Deck 映射**（★=该页独占一页，其余可合并）：

| Deck 页 | 源 BP 段 |
|---|---|
| ★封面/一句话定位 | 1 |
| ★痛点+市场（痛点数字） | 2 |
| ★产品/UVP | 3 |
| 技术/壁垒 | 4 |
| ★市场 TAM/SAM/SOM | 5 |
| 商业模式 | 6 |
| ★财务（现金流转正点） | 7 |
| 里程碑/规划 | 8 |
| ★团队 | 9 |
| 风险+社会价值 | 10+11 |

**② BP 没有、路演必加的两页**：产品现场演示/MVP 页（真演示或录屏，证明"不是纯 PPT 概念"）；融资/资源与合作需求（ASK）收尾页（要多少钱/换什么资源、用途、能跑到哪个里程碑）。

**③ 时长分配（总和卡 5–8 分钟，留 Q&A 缓冲）**：

| 页 | 秒 |
|---|---|
| 封面+痛点 | 45 |
| 产品 UVP | 50 |
| MVP 现场演示 | 70 |
| 市场 SOM | 40 |
| 商业模式+财务 | 60 |
| 团队 | 30 |
| 里程碑+风险 | 35 |
| ASK 收尾 | 30 |
| 合计 | ≈360s(6min)，余 1–2min 缓冲 |

**④ 评委记忆点页**（每页留一个钩子）：痛点数字、UVP 一句话、市场 SOM、财务现金流转正点、团队。

## 2. 互联网+创意组 — 偏早期，重创意与可行性

骨架同上但简化财务（启动资金用途 + 简化盈利模式），更强调：创意新颖度、技术可实现性、最小可行产品(MVP)或原型证据、未来商业化路径。评委容忍"还没盈利"，但不容忍"逻辑不闭环/纯 PPT 概念"。

## 3. 挑战杯"大挑" — 课外学术科技作品（学术向）

三类作品，结构略异，共性是**重研究深度与学术创新**：
- **自然科学类学术论文**：摘要 → 引言(问题与意义) → 国内外研究现状 → 研究方法/实验设计 → 结果与分析 → 讨论 → 结论 → 参考文献。评：选题价值、方法科学性、数据可靠性、创新性、论证严谨。
- **哲社类社会调查报告**：摘要 → 选题背景与意义 → 文献综述 → 调研设计(抽样/问卷/访谈方法) → 调查结果 → 分析与对策建议 → 结论。评：**一手调研数据**、方法规范、现实针对性、对策可操作。臆造样本/数据是硬伤。
- **科技发明制作（A 技术含量 / B 实用新型）**：作品背景与现状 → 技术方案/原理 → 创新点 → 实物/样机与测试数据 → 应用前景 → 知识产权情况。评：技术先进性(A)/实用性与市场(B)、完成度（要有实物或可演示原型）。

**评审维度（公开口径）**：科学性、先进性（创新性）、现实意义 / 应用价值，分初评（书面）与终审（答辩/展示）两阶段。【确切分值待核查当届】

**高分特征**：有真实数据/实物/原型；创新点与现有技术差异化清楚；社会调查类有规范一手调研。
**常见出局点**：当成课程论文交（无创新）；社会调查无一手数据；发明制作只有 PPT 没有实物。

## 4. 大创 / 科研训练 — 申报书（重培养价值与过程）

**通用申报书骨架**（与基金/挑战杯大挑同构，可互借）：
1. 项目名称 + 项目类型（创新训练/创业训练/创业实践）
2. 立项依据与研究意义（知识缺口开场 → 为什么值得做）
3. 国内外研究现状（说清"别人做到哪、还差什么"）
4. 研究内容与目标（具体、可考核；可借 NIH Specific Aims 的"2–4 个彼此独立但互补、且在周期内可完成"的拆法）
5. 技术路线/研究方案（每阶段拆成可验证小步、有交付物，忌笼统"第一阶段调研"）
6. 创新点（与现有方案差异化、可量化、可展示）
7. 可行性分析（技术有基础 / 资源算力数据经费 / 时间里程碑 / 团队分工匹配）
8. 研究基础（已有工作、前期数据，降低评委感知风险）
9. 进度安排（里程碑 + go/no-go 检查点，对齐申报→中期→结题三阶段）
10. 预期成果（论文/软著/专利/实物，具体到数量与形态）
11. 经费预算（见 [budget_template.md](budget_template.md) 第 2 节，逐项对齐研究内容）
12. 团队分工与基础（学生主体，每人职责与能力匹配）

**评审维度（公开口径）**：选题/创新性、方案可行性、**学生主体与培养价值（过程性成长）**、研究基础、预期成果、经费合理性。【确切分值各校不同，待核查】

**高分特征**：研究内容与技术路线一一对应；创新点可量化；预算与研究活动匹配；突出学生主导和能力成长（大创区别于纯科研项目的关键）。
**常见出局点**：创新点空泛；技术路线笼统；预算与内容脱节；像导师课题的"打工版"看不出学生主体。

## 5. 数学建模 CUMCM / MCM-ICM — 论文结构

详见 light-competition SKILL.md 与 references.md 已核实硬规则，要点回顾：
- **CUMCM**：摘要(单独一页、决定初筛，写清模型名+关键结论数字) → 问题重述 → 模型假设 → 符号说明 → 模型建立与求解 → 结果分析与检验 → **灵敏度分析** → 优缺点 → 参考文献 → 附录(完整可复现代码)。两级评审。
- **MCM/ICM**：Summary Sheet 必须第一页且权重极高（忌问题重述/复制引言，最后写）；**全篇 25 页硬上限**（含摘要+正文+参考文献+目录+附录+代码；唯一例外是必填的 "Report on Use of AI" 不计页）；99 小时三人队；奖项 Successful Participant<Honorable Mention<Meritorious<Finalist<Outstanding。

**高分特征**：摘要把方法与最重要结论数字讲透；每问"模型→求解→检验"闭环；有灵敏度分析；MCM 严守 25 页且如实写 AI 说明。
**常见出局点**：CUMCM 摘要写差直接初筛出局、漏灵敏度分析；MCM 误以为附录代码不计页导致超页、Summary 沦为问题重述、漏 AI 说明。

## 6. 跨赛事通用：提交前评审模拟（评委视角自审）

借 light-competition SKILL.md "评审模拟"，逐项过：
- 创新够不够（与现有方案差异化、可量化）？
- 可行不可行（技术/资源/时间/团队四维）？
- 商业逻辑通不通（创业类：UVP 与壁垒、市场自下而上、现金流转正点）？
- 预算合不合理（每笔有测算依据、与活动匹配）？
- 答辩会被问什么（预演 QA，准备备份附录）？
- 跨材料说法是否一致（申报书↔PPT↔预算↔财务预测同一套数，a07 把关）？
- 数据真实性（市场/财务/调研数据可核查、标出处、≤2年；风险上报 a10）？
