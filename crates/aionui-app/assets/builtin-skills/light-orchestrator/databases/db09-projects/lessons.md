# lessons.md — 跨项目过程/方法论教训库

与 `projects/` 平级。定位：**跨项目可复用的过程/做法经验**，新项目立项或选方法/投稿策略前先来检索（见 a02 SKILL.md「会话开始时」）。

## 边界（收什么 / 不收什么）
- **收**：做法层面的有效或失败经验——踩过的坑、被审稿/导师否掉的套路、复现失败的流程、某操作流程显著省时或避雷的经验。
- **不收**：
  - 方法选型「事实」（某方法在某任务上更优等）→ 归 db03 方法卡。
  - 个人偏好（写作风格/工具/格式习惯）→ 归 Light feedback 记忆文件。
  - 单项目内的具体决策（不具跨项目复用性）→ 留在该项目 decision_log。

## 格式（追加式单行，只增不改写）
`- [YYYY-MM-DD] 阶段/场景 — 做法 — 结果(有效|失败) — 适用条件 — 来源项目slug`
