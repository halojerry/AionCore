# Operational Modes（轻量操作模式）

按用户需求选最小够用的工作量，别每次都走全流程。每个模式列：**触发**、**输入**、**步骤**、**产出**、**诚信门档位**。

---

## full —— 完整初稿
- **触发**：用户要"写整篇/一篇完整草稿"。
- **输入**：创新点(m03/m04)、结果(m06)、目标 venue(db01/db02)。
- **步骤**：Paper Configuration → 贡献清单 → 大纲+论证图 → 选模板(templates/) → 认领 reporting 指南(guideline_map) → 逐节初稿(GAP 标记) → 8维+7模式自检 → 填必备声明。
- **产出**：分模块 Markdown/LaTeX 草稿 + 贡献清单 + GAP 台账 + 声明。
- **诚信门**：初稿门(≥30% 抽样)；提交前转终稿门(100%)。

## outline-only —— 只给大纲+论证图
- **触发**："先给我个结构/提纲/论证骨架"。
- **输入**：方向 + venue 类型。
- **步骤**：Paper Configuration → 贡献清单 → 选模板填节序与每节 brief → 画论证图(论点→证据→反驳)。
- **产出**：带每节 brief 与字数预算的大纲 + Argument Map。不写正文。
- **诚信门**：不适用（无 claim）；但 gap 已在 brief 里预标。

## abstract-only —— 只写摘要
- **触发**："帮我写/改摘要"。
- **输入**：背景、问题、方法、关键结果(带数字)、意义。
- **步骤**：套结构(背景→问题→方法→结果→意义)，150–250 词；缺数字标 `[RESULT GAP]`。
- **产出**：一段摘要（+ 可选双语）。
- **诚信门**：摘要内每个数字与每个"SOTA/优于"措辞全查（高危类不抽样）。

## section-redraft —— 重写某一节
- **触发**："把方法/实验/引言这节重写/改写"。
- **输入**：现有该节文本 + 该节 brief（要论证什么）。
- **步骤**：定位该节模板要点 → 对照 brief 重写 → 该节级 8维+7模式自检 → GAP 标记。
- **产出**：该节新稿 + 改动说明（动了什么、为什么）。
- **诚信门**：对该节 claim 抽样≥30%；触及结果数字则全查。

## self-review —— 只做自检/模拟审稿
- **触发**："帮我审一下/挑毛病/模拟审稿"。
- **输入**：已有草稿。
- **步骤**：跑 self_review_checklist（8维+7模式逐项）→ 按 integrity_gate 抽样核 claim 与引用 → 可选机检 `scripts/draft_lint.py`。
- **产出**：逐维问题清单 + 命中的失败模式 + 必改项（触发改写列）+ 声明缺失项。
- **诚信门**：按所处阶段定档（初稿门或终稿门）。

---

## 模式选择速判
- 不知道写啥结构 → outline-only。
- 已有大纲要铺全文 → full。
- 只缺某块 → section-redraft / abstract-only。
- 写完想被挑刺 → self-review。
- 任何模式产出，凡未核实内容一律 `[MATERIAL GAP]`/`[RESULT GAP]`，绝不凭空填充。
