# Self-Review Checklist（可勾选）

每个模块写完立即过一遍。两部分：**A. 8 维质量自检**（ScholarEval 维度）、**B. 7 类 AI 失败模式红线**（源自 The AI Scientist 失败模式分析）。
每项两栏：**看什么**（检查动作）/ **什么触发改写**（命中即返工，不是降分了事）。

---

## A. 8 维质量自检

| # | 维度 | [ ] | 看什么 | 什么触发改写 |
|---|---|---|---|---|
| A1 | 创新性 | [ ] | 贡献清单是否 punchy 且与已有工作有明确 delta | 说不出"比 X 多了什么"，或贡献是已知结论的换皮 |
| A2 | 意义/影响 | [ ] | 谁会用、解决什么真实痛点 | 只能答"有趣"，答不出"对谁有用" |
| A3 | 方法严谨性 | [ ] | 假设显式、推导/流程无跳步、baseline 公平 | 出现"易证/显然"跳步，或 baseline 不可比/调参不公平 |
| A4 | 清晰度 | [ ] | 术语首现即定义、图表自解释、段落有主句 | 读者需回看才懂某句，或图表脱离正文看不懂 |
| A5 | 可复现性 | [ ] | 数据/超参/环境/随机种子/代码可得 | 缺 seed、缺关键超参、"按标准设置"含糊 |
| A6 | 实验证据支撑 | [ ] | 每个 claim 有对应数据；含显著性与方差 | claim 无数据支撑，或只报单次结果无 std/CI |
| A7 | 与已有工作定位 | [ ] | Related 每类点明差异、主表含强 baseline | 流水账综述，或回避最强对手方法 |
| A8 | 伦理与局限诚实度 | [ ] | Limitations 真诚、不利结果不藏、声明齐全 | Limitations 是套话，或藏了不利消融/失败 case |

## B. 7 类 AI 失败模式红线

| # | 失败模式 | [ ] | 看什么 | 什么触发改写 |
|---|---|---|---|---|
| M1 | 实现 bug 当通过 | [ ] | 结果好得反常时，回查实现与数据泄漏 | 指标异常高/无 train-test 隔离/评测脚本未核 → 重跑前不得写入正文 |
| M2 | 幻觉引用 | [ ] | 每条引用的 DOI/arXiv 是否 curl 实测可达、题名作者年份一致 | 任一引用查无此文或字段对不上 → 删除或替换，绝不"大概是这篇" |
| M3 | 幻觉实验结果 | [ ] | 正文每个数字是否来自实际运行/可核的他人原文 | 数字凭语感填、或他人数字未核原文 → 改标 `[RESULT GAP]` 待补 |
| M4 | 走捷径 | [ ] | 是否用简化设定冒充一般结论、是否挑数据 | 只在 toy/单数据集成立却写成普遍结论 → 限定适用范围 |
| M5 | bug 包装成新洞察 | [ ] | "意外发现"是否先排除了 bug/泄漏/统计假象 | 解释建立在未排查的异常上 → 先证伪 bug 再谈洞察 |
| M6 | 方法学造假 | [ ] | 流程描述与实际所做是否一致、有无事后编造步骤 | 写了没做的步骤/编造的协议 → 改写为实际所做 |
| M7 | 框架锁死(frame-lock) | [ ] | 是否早早锁死叙事、对反例/反驳视而不见 | Devil's Advocate 反驳打分≥4 仍不让步 → 改写贡献而非硬撑 |

---

## 用法
- 初稿：A 全过 + B 全过；命中"触发改写"列即返工对应模块。
- 引用核查不可只看 citation key——key/DOI 只是溯源字段，不等于该文真支撑论点；存疑的点开原文核对支撑关系（对接 M2/M3）。
- 配合 `integrity_gate.md` 的抽样配额执行；可用 `scripts/draft_lint.py` 机检 GAP 标记与声明缺失，但人判优先。
