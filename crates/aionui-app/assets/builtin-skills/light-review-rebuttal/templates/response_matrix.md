# 回应矩阵 · {{论文标题 / venue + 轮次}}

> 用法：每条审稿意见一行，**先填左半（盲读意见+分类+concession 自评）再填右半（回应+改动）**。
> concession 分（1–5，借反谄媚口径）：5=撤回原观点 / 4=降级 / 3=保持(给新证据) / 2=重述(无新料) / 1=加强。
> **让步必须挂新证据/新实验**；禁连续让步（相邻两条都 ≥4 需第二条独立证明，否则按 ≤3）。

## 分类取值（与 m08 findings/严重度口径一致）
- 分类：实验 / 创新性 / soundness / 写作 / 可复现 / 相关工作 / 其他
- 严重度：major（影响录用，必处理）/ minor（建议性）/ question（澄清即可）
- re-review 判定：**已改可核**（改了且能在稿里指到位置）/ **仅回应不改稿**（礼貌反驳，给理由）/ **部分采纳**（说明改了哪部分）

## 闭环自检（提交 rebuttal 前，配合 rereview_checklist.md）
- [ ] 每条 major 意见都有"改动位置"或充分的"不改稿理由"，无遗漏。
- [ ] 所有 concession ≥4（撤回/降级）都挂了**新证据/新实验**，非空口让步。
- [ ] 无连续让步（相邻 ≥4 的第二条已独立证明，否则降为 ≤3）——跑 `verify` 思路或人工核。
- [ ] "已改可核"行的改动位置在修订稿里**真能指到**（不是承诺了没改）。
- [ ] 引用了新文献的回应，已过 m10 verify_refs/citekey_audit（新 \cite 进 .bib）。
- [ ] rebuttal 文本字数/字符在 venue 预算内（跑 `scripts/rebuttal_budget.py --venue <会>`）。
