# 自检清单（交付前逐项打勾）

> 复制到任务收尾处，逐项判 ✓/✖/!。任何 ✖ 未清不许交付。
> 配套：references/excuse_intercept.md（借口拦截）、self_check_contrasts.md（反例对照）、receiving_feedback.md（接收反馈）。

## 0. 证据闸门（先过这关）
- [ ] 我要下的「完成/通过/修好」结论，有**当前轮当场跑出**的证据，不是旧结果或推测
- [ ] 已读完整输出、查过 exit code、数过失败条数
- [ ] 输出确实支撑结论（不支撑就如实报真实状态）
- [ ] 没有出现红旗词 should/probably/seems to / 提前自我满足 / 轻信子代理报告

## 通用 11 项（判 ✓/✖/!，对照 self_check_contrasts.md）
- [ ] 1 逻辑：论证链无断点、跳跃、自相矛盾
- [ ] 2 事实：数据/引用/声称真实可核查，无臆造（CONVENTIONS §4）
- [ ] 3 格式：符合模板/字数/引用风格/编号
- [ ] 4 表达：清晰、专业、无歧义、无口语
- [ ] 5 创新：贡献突出，非常规堆叠（联动 m04）
- [ ] 6 引用：准确、相关、不遗漏、不堆砌（联动 m10）
- [ ] 7 夸大：结论不超出证据；慎用 novel/significantly/SOTA
- [ ] 8 审美：图表/排版/PPT 风格统一（联动 a07）
- [ ] 9 重复：无冗余内容、无重复段落
- [ ] 10 结构：组织合理、层次清晰
- [ ] 11 可执行：方案/代码/步骤当场跑通、能落地

## 借口自查（对照 excuse_intercept.md）
- [ ] 扫过借口拦截表，没有用「改动小/应该能过/上次跑过/先交了」等借口跳过验证
- [ ] 没有为缩小证据范围而自我放行（主路径通≠完成，大部分过≠通过）

## 按产出类型补充
- [ ] 代码：能运行；新功能/改 bug 走过红-绿（撤掉修复看到变红，恢复转绿）；依赖安全（无可疑装包脚本/typosquatting）
- [ ] 论文/材料：结论被结果支撑（评审第一失败点）；方法学扎实；文献根基实；格式合规（联动 m14）
- [ ] 数据/结果：统计站得住（联动 m06）；对照/样本/漂移逐项核（Deepchecks/Evidently 三态）
- [ ] 合规：无伦理/版权/隐私风险（联动 a10）
- [ ] 若产出是 skill/MCP：查 prompt injection、数据外泄、含糊触发描述、跨技能重叠（记住「无发现≠无威胁」）

## 接收反馈时（对照 receiving_feedback.md）
- [ ] 没有表演式同意（同意带证据，反驳带证据）
- [ ] 不清楚就停下要澄清，没用猜测继续
- [ ] 范围只覆盖被要求的，没顺手重构无关代码（YAGNI）

## 失败循环
- [ ] 同一问题没有反复打补丁；修 ≥2 次仍不对已诊断根因、修 ≥3 次已停手质疑架构

## 收尾
- [ ] 凡判「非问题/可放行」（✓ 或降级）的项，都附了一行带证据的放行理由（命令输出/原文行号/检查结果），没凭印象放行
- [ ] 不通过项已按 Critical(立即修)/Important(交付前修)/Minor(记录待办) 处理
- [ ] 致命问题排在格式小问题之前，没陷入「只挑表述」的表面陷阱
- [ ] 无法自行解决的重大问题已如实告知用户，未隐瞒
