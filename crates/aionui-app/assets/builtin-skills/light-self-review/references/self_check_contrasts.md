# 11 项通用自检：最小 通过✓ / 不通过✖ 对照

> SKILL.md 通用自检清单的每一项，配一组最小可判别的反例对照。
> 用法：自检时不写「感觉还行」，而是对照本表判 ✓ / ✖ / ! 三态，命中 ✖ 即按严重级修。

---

### 1. 逻辑（论证链有无断点、跳跃、自相矛盾）
- ✖ 「A 与 B 相关，所以 A 导致 B。」——相关跳因果。
- ✓ 「A 与 B 相关（r=.4）；因果需干预实验，本文不下因果结论。」

### 2. 事实（数据/引用/声称是否真实可核查，有无臆造）
- ✖ 「据 Smith (2021) 准确率达 99%。」——文献/数字未核，疑似臆造。
- ✓ 「Smith (2021, DOI:10.x/y，已 curl 200) 报告 91.3%（Table 2）。」

### 3. 格式（是否符合模板/字数/引用风格/编号要求）
- ✖ 正文 5200 字（要求 ≤5000），引用混用 GB/T 与 APA。
- ✓ 4870 字，全文 APA 第 7 版，图表编号连续无跳号。

### 4. 表达（清晰、专业、无歧义、无口语）
- ✖ 「这个方法贼好用，效果杠杠的。」——口语、无信息。
- ✓ 「该方法在 X 数据集上较基线降低 12% 错误率。」

### 5. 创新（贡献是否突出、不是常规堆叠）
- ✖ 「我们把 A 模块换成 B，再加个注意力。」——模块堆叠无洞见。
- ✓ 「我们指出现有方法在 Z 假设下失效，提出 W 机制解决（联动 m04）。」

### 6. 引用（准确、相关、不遗漏、不堆砌）
- ✖ 一句话挂 12 篇引用，其中 3 篇与论点无关。
- ✓ 关键主张各配 1–2 篇直接支撑文献，最相关的奠基工作未遗漏（联动 m10）。

### 7. 夸大（结论是否超出证据支撑）
- ✖ 「本方法显著优于所有方法，达 SOTA。」——无显著性检验、未全比。
- ✓ 「在 2/3 基准上优于基线（配对 t 检验 p<.05），另一基准持平。」

### 8. 审美（图表/排版/PPT 风格是否统一）
- ✖ 三张图配色字号各异，术语一图 "acc" 一图 "准确率"。
- ✓ 统一配色板、字号、术语；图注格式一致（联动 a07）。

### 9. 重复（有无冗余内容、重复段落）
- ✖ 引言与结论几乎逐句重述方法贡献。
- ✓ 每处信息只在最合适处出现一次，交叉引用代替复制。

### 10. 结构（组织是否合理、层次是否清晰）
- ✖ 方法细节散落在结果与讨论里，读者需来回翻找。
- ✓ 标准层次：动机→方法→实验→分析；每节单一职责。

### 11. 可执行（方案/代码/步骤是否真能跑通、能落地）
- ✖ 「运行 main.py 即可」——缺依赖说明、路径写死、未实测。
- ✓ 当前轮 `python main.py` exit 0、给出环境与数据准备步骤（联动 a03/证据闸门）。

---

## 三态判定速记
- **✓ 通过**：有当场证据支撑，符合要求。
- **✖ 不通过**：命中上面任一反例形态 → 附严重级 Critical/Important/Minor 并修。
- **! 警告**：无法当场证实也无法证伪 → 如实标「需核查」，不放行为 ✓。
