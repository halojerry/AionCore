# 自检走查示例（证据闸门 + 借口拦截 + 三态/严重级的一次完整应用）

下面两个端到端示例，演示证据闸门循环如何在真实物料上跑完一遍：
草稿 → 命中借口 → 当场重跑/核查 → ✖ 判定 + 严重级 → 修 → 重新验证拿到新鲜证据 → 交付。
配合 SKILL.md 的证据闸门、`excuse_intercept.md`、`self_check_contrasts.md` 阅读。

## 示例 A：代码（命中借口 #5 / #10）

**草稿结论**：「编译过了、测试我跑过大部分都绿，已修复，交付。」

1. **借口拦截命中** → #5「编译通过≠行为正确」、#10「大部分通过≠0 失败」。命中即回到证据闸门，不采信该结论。
2. **证据闸门**：当前轮重跑 `pytest -q`（不引用上次结果）→ 读完整输出：`12 passed, 2 failed`，exit code 1。
3. **三态判定**：✖ 不通过。严重级 = **Critical**（测试红 = 阻塞交付）。
4. **修**：定位 2 个失败用例 → 改代码。
5. **红-绿核验**：撤掉修复重跑，确认该用例确实变红（证明测试有效）；恢复修复后重跑 → `14 passed, 0 failed`，exit 0（当前轮新鲜证据）。
6. **交付**：附「当前轮 pytest：14 passed / 0 failed / exit 0」。结论由新鲜证据支撑，非空口。

## 示例 B：论文段落（命中借口 #14）

**草稿句**：「本方法 significantly 优于所有基线，达到 SOTA。」

1. **借口拦截命中** → #14「无显著性检验不写 significant」；通用清单「夸大」项触发 ✖。
2. **核查**：查证据 → 未做配对显著性检验，且只比了 3 个基线中的 2 个。结论超出证据支撑。
3. **三态判定**：✖ 不通过。严重级 = **Important**（交付前必须修，否则审稿直接质疑）。
4. **修**：补配对 t 检验后，据实降级表述 → 「在 2/3 基准上优于基线（配对 t 检验 p<.05），第三个基准持平」。
5. **重新验证**：对照「夸大」项反例 → 现表述与证据一致 → ✓ 通过。
6. **交付**。

## 严重级排序（呼应 SKILL.md 与 receiving_feedback §6）

两例都把**致命问题（Critical/Important）排在格式类 Minor 之前**先处理：
先修「测试红 / 结论无证据」，再处理「标点、编号、措辞」等表面项——
不陷入「只挑格式」的表面陷阱。
