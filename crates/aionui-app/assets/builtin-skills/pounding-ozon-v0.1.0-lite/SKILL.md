# pounding-ozon 执行规约

## 什么时候使用本技能

当用户要做以下事情时，Agent 应当使用本技能：

- 从 1688 选品并继续发布到 Ozon
- 批量铺货
- 运行单商品完整发布链路
- 翻新已有 Ozon 商品
- 查询铺货 / 翻新 / 发布任务状态
- 做采购询盘后继续回到本项目执行链路

本技能不负责介绍项目原理，不负责复述代码里已经硬编码的流程，不负责把内部实现讲成 README。

## 严格禁止

- 禁止绕过 `cli.py` / `scripts/capabilities/*` 手工拼流程
- 禁止在脚本失败后凭主观判断改走另一套隐式流程
- 禁止依赖参考项目目录运行
- 禁止把 dry-run 说成真实上架成功
- 禁止把“上传任务已提交”说成“Ozon 已最终过审”
- 禁止编造脚本未返回的字段
- 禁止把询盘说成下单
- 禁止把测试产物、临时路径、调试目录当成发布版前提

## 命令选择

用户意图 → 命令映射：

- 只补配置 / 查配置：`configure`
- 文字找货：`text_search`
- 以图找货：`image_search`
- 1688 链接找货：`link_search`
- 候选对比：`compare`
- 采购询盘：`procurement`
- 完整发布链路：`publish_flow`
- 老商品翻新：`refresh_product`
- 任务 / 商品状态查询：`status`

默认原则：

- 只要用户目标是“从选品到上架 / 翻新”，优先走 `publish_flow` 或 `refresh_product`
- 不要把搜索、类目、属性、图片、上传拆成多段手工编排，除非脚本返回明确要求

## 变量收集规则

优先从当前输入里抽取：

- `query`
- `image`
- `url`
- `draft-file`
- `drafts-file`
- `assets-file`
- `task_id`
- `product_id`
- `offer_id`
- 是否 `dry-run`
- 是否 `poll-status`

缺变量时的顺序：

1. 先从当前对话和文件里推导
2. 再看能否先调用现成命令拿到下一步变量
3. 只有关键变量仍缺失时，才问用户

不要替用户臆造未提供的业务输入。

## 配置检查规则

真实执行前必须确认这四项存在：

- `ALI_1688_AK`
- `OZON_CLIENT_ID`
- `OZON_API_KEY`
- `MXOU_TOKEN`

缺失时：

- 立即停止真实执行
- 先走 `configure`
- 返回缺失项和下一步

## 执行规则

- 必须优先调用 `cli.py` 暴露的能力
- 必须依赖 `scripts/capabilities/*` 现有链路
- 必须先读结构化返回，再决定下一步
- 必须把“已提交”“审核中”“已上架成功”区分开
- 必须在完整链路结束后返回货源链接、成本、利润率、建议售价、任务结果

批量任务要求：

- 必须形成队列
- 默认自动并发
- 小批量自动 `2` 并发
- 大于等于 `4` 个商品时自动 `3` 并发
- 不要让并发无限放大

如果脚本返回批量结果，必须按条目汇报：

- `source_item_id`
- 1688 来源链接
- `task_id`
- 成功 / 阻断 / 失败

## 失败处理

发生失败时，不要脱离程序直接给业务结论。

必须按顺序：

1. 读 `error_code`
2. 读 `details`
3. 读 `data` / `stages`
4. 判断失败类型
5. 选择最小下一步动作

常见失败分类：

- 缺配置
- 缺关键变量
- 候选过多需确认
- 发布门禁阻断
- 外部 API 暂时失败
- 状态仍处理中

## 输出规范

### 单商品完整链路结束后，必须返回

- `source_url` / `source_1688_url`
- 1688 来源链接
- `offer_id`
- `product_id`
- `ozon_task_id`
- `procurement_cost`
- 采购成本
- `profit_margin`
- 利润率
- `suggested_price`
- 建议售价
- 当前状态

并给出简短业务摘要：

- 货源链接
- 上架结果
- 成本
- 利润率
- 建议售价
- 当前审核 / 处理状态

### 批量任务

先给批量总览：

- 总数
- 成功数
- 阻断数
- 失败数
- 队列模式
- 最大并发

再给每个商品的结构化结果。

## Honesty rules

- 只对脚本真实完成的阶段宣称成功
- 处理中就说处理中
- 被阻断就说阻断原因
- 未返回字段就不要补写

## 发布包假设

默认用户运行的是发布版：

- 不要依赖工作区临时路径
- 不要依赖测试产物
- 不要依赖参考项目
- 不要要求用户自己手动拼内部文件

本 skill 的职责就是把 Agent 约束回“用户给目标 → 调本项目现成能力 → 返回结构化业务结果”。
