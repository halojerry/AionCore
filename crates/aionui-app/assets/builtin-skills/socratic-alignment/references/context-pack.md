# Context Pack 模板

Context Pack 是给 Agent 的“任务地图”。

没有 Context Pack，用户丢给 Agent 的往往只是一个愿望；有了 Context Pack，Agent 才知道目标、边界、素材、验收标准和不能碰的地方。

## 标准字段

```markdown
# Context Pack

## 背景
{这件事从哪里来，为什么现在讨论}

## 真实意图
{用户真正想解决的问题，不是表层任务}

## 刺点
{当前最大的矛盾 / 悖论 / 单点故障}

## 目标用户 / 场景
{谁，在什么场景下，为什么痛}

## 触发场景
{用户什么时候会主动使用？什么时候最需要但不会使用？}

## 当前判断
{用户目前相信什么}

## 关键假设
1. 需求假设：
2. 触发假设：
3. 付费假设：
4. 交付假设：
5. 差异化假设：

## 已知证据
- {事实 / 观察 / 用户访谈 / 数据 / 个人经验}

## 未知问题
- {还没有证据的问题}

## 约束
- 时间：
- 钱：
- 能力：
- 不想做的事：

## 非目标
{明确不做什么，防止 Agent 发散}

## 正例
{什么样的输出算对}

## 反例
{什么样的输出一看就跑偏}

## 验收标准
{用户如何判断 Agent 输出可用}

## 人类判断门
{哪些结论必须由人决定}
```

## 使用原则

Context Pack 不是越长越好。  
它要足够具体，能减少 Agent 猜测；也要足够克制，不把无关信息塞进去。

读完它，Agent 应该能回答三句话：

1. 我到底要帮用户推进什么？
2. 我不能替用户判断什么？
3. 我什么时候必须停下来问人？
