# 苏格拉底式追问卡

苏格拉底式追问不是审问用户，也不是故意抬杠。

它的作用是：**用一个有本质的问题，让用户自己看见自己的前提。**

## 提问铁律

1. 每轮只问 1 个问题。
2. 优先使用 `AskUserQuestion` 工具。
3. 问题要有取舍、有反差、有刺点。
4. 给选项，降低回答成本。
5. 用户不想回答，就按默认假设推进。

## 坏问题和好问题

| 坏问题 | 为什么坏 | 好问题 |
|---|---|---|
| 你的目标用户是谁？ | 太像表单 | 第一批会疼到打开它的人是谁？别说“创作者”，给一个具体人。 |
| 你有什么资源？ | 信息有用但不锋利 | 这件事里，哪个环节最可能把一个人拖死？ |
| 你什么时候会用？ | 太平 | 最需要它的时候，用户大概率不会主动打开它。这个悖论你怎么破？ |
| 你担心什么？ | 太宽 | 如果这只是一个漂亮概念，最可能漂亮在哪里？ |
| 怎么验证？ | 太普通 | 72 小时内，哪个信号能证明“判断力陪练”不是自嗨？ |

## 五类高价值问题

### 1. 悖论问题

- 你说用户缺判断力，但缺判断力的人会主动买判断力吗？
- 最需要这个搭子的时候，用户会不会想起用它？
- 这东西是在帮人清醒，还是把 AI 的答案包装得更像清醒？

### 2. 刺痛问题

- 用户到底是痛在“不会做”，还是痛在“不敢判断”？
- 用户愿意为这个付费，是为了得到答案，还是为了少走弯路？
- 哪个场景里，用户不用它会真的付出代价？

### 3. 反证问题

- 如果这个方向失败，最可能不是因为没需求，而是因为什么？
- 哪个前提一旦不成立，整件事就垮？
- 你最不愿意承认的反面证据是什么？

### 4. 认知缴械问题

- 这个判断是你自己想明白的，还是 AI 帮你说圆的？
- 你是不是把“思想成立”误当成“产品成立”？
- 哪一步是你必须亲自拍板，不能交给 Agent 的？

### 5. Agent 对齐问题

- Agent 执行前，哪个上下文没说清会直接跑偏？
- 这个任务的反例是什么？什么输出一看就不对？
- Agent 到哪一步必须停下来等你判断？

## AskUserQuestion 模板

```markdown
问题：{一个锋利问题}
为什么问：{一句话说明它决定什么}
选项：
A. {方向一}
B. {方向二}
C. {方向三}
D. 先按 {默认假设} 推进
```

## 提问原则

问题要像一把小刀，切开一个关键矛盾。  
不要像问卷，一次铺开十个字段。
