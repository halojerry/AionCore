# 成长成功率：管理层与治理质量（VC 透镜）

> 本文件按需加载。SKILL.md 主流程在分析成长股、或确认 B 型「未被定价的期权」时引用。

## 0. 为什么成长股要单独看这一层

成长股的估值，本质是**对未来预期的贴现**——它相对成熟公司多出来的那块溢价，不是为已经发生的利润付的钱，而是**一笔赌注：赌管理层能把财务还证明不了的成长 / 新方向兑现出来**。

一级市场 VC 早就在做这件事。早期公司没有财务可依靠，VC 只能去评估创始团队、创始人素质、公司治理历史，借此判断"这家公司开拓新方向的成功概率"。二级市场的成长股溢价同理——区别只是这里有财务数据、有公开披露、有可查的历史记录，反而能把这套判断做得**比 VC 更有据可依**。

把它接到本 skill 已有的概念上：B 型「未被定价的期权」（见 `deep_mode.md §5`）的期望价值 ≈ **赔付 × 兑现概率 P(execution)**。骨架分析擅长估赔付（TAM、远期空间、渗透率），却把 P(execution) 当成隐含且均匀的常数。本框架就是 **P(execution) 的显式估计器**：用"好公司 + 好团队"的成色，给成长 / 期权的兑现概率定一个档位，再把这个档位量化接回估值（§4）。

一句话：**赔付回答"如果成了能值多少"，成长成功率回答"凭这帮人、这套治理，成的概率有多大"。** 两者缺一，成长股估值就是半截子。

## 1. 何时启用，权重怎么给

- **成长股 / 确认了 B 型期权的票 = first-class**：本框架是成长逻辑能否成立的核心一环，必须做，并按 §4 接回定量。
- **成熟龙头 / 红利价值 = 防御性轻用**：治理 / 团队数据回到"背景与风险信号"的角色（管理层激励、大股东增减持、一言堂风险），不必跑完整三子框架、不接概率权重。理由：这类公司的价值由已兑现的盈利和现金流支撑，对"未来新方向的成功率"不敏感。
- **类型切换期**（成长股增速降档、或成熟龙头被新主线点燃要讲新故事）：只要估值里含有"未兑现的新方向溢价"，就该做这一层。

## 2. 子框架（A/B/C 三要素 + D 组织层）

A / B / C 对应最初提出的三要素（创始人素质、团队历史、治理水平），**D 是补上的组织执行层**——前三者看的都是"顶层的人"，D 回答"底下这台机器能不能跑"。每个子框架都给"要回答什么 + 数据源 + 绿旗 / 红旗 + 骨架可得 vs Deep 才挖得到"。骨架模式用已取数据（`company`/`managers`/`rewards`/`top10`/`holder_trade`/`income`）做初判；Deep 模式按 `deep_mode.md §方向三` 读公告 / 年报 + 联网把它做实。综合判断时还可叠加 §2.5 的几条横切透镜。

### A. 创始人 / 掌舵人素质 ——「掌舵的是不是对的人」

要回答：开拓新方向的这帮人，有没有把新方向做成的底子和动机。

- **数据源**：`company`（实控人）、`managers`（董监高名单、职务、`birthday`、`begin_date`/`end_date`、学历）、`rewards`（持股 `hold_vol`、薪酬 `reward`）、联网（创始人履历、过往言论、行业声誉）。
- **绿旗**：founder-led 且创始人技术 / 产业出身；核心高管多年稳定、关键岗位（董事长 / 总经理 / 财务总监 / 董秘 / 核心技术负责人）少变动；管理层公开承诺与后续兑现一致；核心团队持有可观股份、与小股东同船。
- **红旗**：实控人是财务投资人 / 接盘方而非产业家；核心岗位（尤其 CFO、核心技术）近 3 年频繁换人；过度依赖单一灵魂人物且无梯队（key-man risk）；管理层几乎不持股、薪酬与公司规模 / 业绩明显脱节；创始人已淡出而新方向却由他当年的故事支撑。
- **创始人尾部风险（A 股特有，必扫一遍）**：实控人高比例股权质押逼近平仓线、离婚分割股权、对赌 / 协议转让平仓、同时实控多家上市公司导致精力分散、健康 / 年龄与接班空缺——这些"人"层面的尾部事件能瞬间击穿成长逻辑（与 C 的质押 / 减持交叉印证）。
- **骨架可得**：高管稳定性（begin/end）、年龄梯队（birthday）、持股 / 薪酬绑定（rewards）。**Deep 才挖**：创始人履历、连续创业战绩、行业口碑、言行一致性（联网 + 公告）。

### B. 历史团队表现 / track record ——「他们以前说到做到吗」（最硬的信号）

要回答：这支团队过去许下的承诺、画过的饼，兑现率如何。承诺兑现度是整个框架里**信息密度最高、最难造假**的一项。

- **数据源**：股权激励 / 业绩承诺 / 重组公告（考核目标）、后续定期报告（实际达成）、机构调研历史指引、历史财务（ROIC 趋势、商誉减值）、募投进展披露。
- **承诺兑现度（first-class）**：把历次**股权激励考核目标、定增 / 重组业绩承诺、机构调研里给的指引**逐条拎出来，对照后续真实财报——达成 / 踩线 / 落空。屡次达成 = 强绿旗；屡次落空或刚好"精准达标"后变脸 = 强红旗。
- **战略执行历史**：过去开拓的第二曲线 / 新业务，真正做成收入的有几个？还是反复换风口、年年讲新故事却不落地（"PPT 公司"）？
- **资本配置历史**：历次募资投向有没有产生回报（看 ROIC 趋势、募投项目达产率）？并购是变成了利润还是变成了商誉减值？是否圈钱后频繁变更募投用途、或资金长期趴在账上理财？
- **穿越周期能力**：行业下行期是守住份额、还是乱扩张乱并购、或被动收缩。
- **骨架可得**：ROIC / 商誉减值的财务痕迹（已在 thesis_scan 财务旗标里）。**Deep 才挖**：承诺兑现核对、募投达产、并购复盘——必须读激励 / 承诺公告 + 对照财报。

### C. 公司历史治理水平 / 诚信与对小股东态度 ——「他们会不会坑我」

要回答：这家公司过去有没有损害过小股东、有没有诚信污点。再好的成长故事，治理有硬伤都要大幅折价。

- **数据源**：`announcements`（监管函 / 问询函 / 处罚 / 会计差错更正 / 关联交易 / 担保 / 质押 / 减持）、`holder_trade`（增减持节奏）、`top10_holders`（集中度 / 制衡）、`rewards`（激励考核宽松度）、业绩预告 vs 实际。
- **信息披露与合规**：有没有监管函 / 问询函 / 行政处罚 / 会计差错更正历史？业绩"变脸"（预告与实际大幅背离）是否频繁？
- **关联交易 / 资金占用 / 违规担保**：有没有大股东占款、利益输送、违规对外担保的痕迹。
- **对小股东态度**：长期分红 vs 一味圈钱；大股东减持节奏（高位精准减持、清仓式减持 = 红旗）；控股股东**股权质押比例**高 → 既反映财务紧张，也放大掏空动机。
- **治理结构**：一股独大且无制衡（第一大远超第二大，且无机构 / 战投牵制）vs 有制衡；董事会独立性；股权激励是真绑定还是考核目标宽松到形同虚设（走过场）。
- **骨架可得**：集中度 / 制衡（top10）、近期减持（holder_trade）。**Deep 才挖**：监管函 / 处罚 / 资金占用 / 质押比例 / 变脸历史——读公告 + 联网。

### D. 组织能力与人才密度 ——「好船长，配的是不是能打的船员」

要回答：除了掌舵人，整个组织有没有能力把硬骨头啃下来。A/B/C 看的都是顶层，但硬科技、长研发周期的成长，兑现靠的是**组织的执行密度**——空有好船长、底下没人，照样开不动。这一层对高研发、靠工程化落地的成长股权重很高。

- **数据源**：`company`（`employees` 员工数、主营 / 简介）、`income`（`rd_exp` 研发费用）、年报（研发人员数量 / 占比 / 学历、**核心技术人员**名单与变动、员工持股 / 激励覆盖人数）、`rewards`（激励是否只到高管）、募投进展披露。
- **人才密度**：研发人员数量与占比、学历结构；**核心技术人员是否稳定**（年报专门披露其变动——硬科技公司核心技术人员离职是重大红旗）；人均创收 / 人均薪酬的水平与趋势（`营收 / employees`）。
- **激励的广度与进取性**（深化 C 的"激励真伪"）：股权激励 / 员工持股**覆盖到核心骨干上百人**（绑定组织）还是**只绑高管十几人**（绑定个人）；考核目标相对历史增速 / 市场预期是**进取**还是**注水**（轻松达标）；是否**反复下调考核目标 / 调低行权价 / 重新授予**——这是激励诚意崩坏的强红旗。
- **组织与产能配套**：募投项目 / 产能 / 产线 / 客户认证的投放节奏与执行——屡屡延期、达产率长期不及预期，说明组织执行力配不上战略野心。
- **绿旗**：研发人员占比高且稳定、核心技术人员零流失、激励覆盖到骨干、考核目标进取且历次达成、募投按节奏达产、人均创收行业领先且上行。
- **红旗**：核心技术人员流失 / 频繁变动；激励只绑高管或反复注水降标；募投长期延期 / 变更；人均创收显著低于同业或持续下滑；扩张靠堆人头而非提效。
- **骨架可得**：员工数、研发费用率、人均创收（已取数据可算，`thesis_scan` 的 `people_governance.organization` 已摆出）。**Deep 才挖**：研发人员结构、核心技术人员变动、激励覆盖广度与考核进取性——读年报 + 激励公告。

### 2.5 更多判断透镜（A/B/C/D 之外，按需叠加）

不是每家都要跑满，但下面几条对成长股的成功率判断常有边际信息，尤其在判 B 型期权时：

1. **创始人–赛道匹配（founder-market fit）**：这帮人对这条赛道是有"挣来的认知"（深耕多年、技术 / 产业积累与新方向同源）还是半路跨界蹭风口？匹配度高的团队开拓新方向成功率系统性更高；半路转型讲故事的，期权要折价。
2. **第二曲线的邻接性**（直接修正 B 型期权计入比例）：新方向是**内生邻接**（复用现有技术 / 客户 / 渠道 / 产能）还是**外延跨界**（与主业无关的并购 / 改名蹭概念）？内生邻接的期权兑现率远高、可观计入；跨界期权默认按"故事"折价，除非有硬证据。
3. **外部聪明钱的背书**：有没有顶级机构 / 产业龙头 / 战投真金白银入股、大客户深度绑定、被巨头供应链 / 生态纳入？成熟外部方做过尽调还下注，是 P(execution) 的独立交叉验证。但要甄别：pre-IPO 突击入股、关联方 / 纯财务套利不算背书。
4. **预期管理与沟通质量**（区别于 C 的合规）：管理层指引的精准度（历次业绩预告 vs 实际偏差）、机构调研里是开放给信息增量还是回避难题只报喜。沟通越透明、指引越可信，你对其关于未来的叙述就越少打折；反之，所有远期承诺都要重折现。

## 3. 综合成档位（锚点化）

把 A / B / C / D（必要时叠加 §2.5 透镜）的证据综合成一个**成长成功率档位**。档位是**可被推翻的假设**，必须标注置信度，并指明缺了哪类证据。下面三档是锚点（"高成功率意味着……具体长这样"），不要用"中等"这种漂移词：

| 档位 | 锚点：典型信号组合（A/B/C/D 同时满足大部分） | 含义 |
|---|---|---|
| **高成功率 / 可信赖（好公司好团队）** | founder-led 且产业背景扎实、核心团队多年稳定（A 绿旗）；历史承诺基本兑现、募资有回报、无失败并购（B 绿旗）；零重大治理瑕疵、利益深度绑定、有制衡（C 绿旗）；组织人才密度足、核心技术人员稳定、激励覆盖到骨干（D 绿旗）；第二曲线为内生邻接、有聪明钱背书 | 新方向兑现概率显著高于均值，溢价有人撑得起 |
| **中性 / 混合** | 有明显亮点也有明确疑点：如团队强但有过一次承诺落空；或治理干净但创始人已淡出、接班人未验证；或绑定到位但有一次募投变更；或第二曲线为跨界、外部背书弱、指引可信度一般 | 兑现概率约等于行业均值，溢价要打问号 |
| **低成功率 / 存疑（PPT 公司或治理瑕疵）** | 实控人财务化 / 频繁套现；核心岗位动荡、key-man 风险高（A 红旗）；屡次承诺落空、年年换风口不落地、募投变更 / 商誉爆雷（B 红旗）；有监管函 / 资金占用 / 高质押 / 清仓减持（C 红旗，任一硬伤即可触发）；核心技术人员流失、激励反复注水（D 红旗）；第二曲线纯跨界讲故事 | 新方向大概率讲故事，溢价应被大幅折价甚至清零 |

判断纪律：
- **C 的硬伤（资金占用、重大违规、监管处罚、控股股东超高质押 + 清仓减持）可单独把档位压到"低"**，即便 A / B / D 看着不错——治理污点是一票否决项，不被业务故事对冲。
- A / B / D 的优势可以互相补强，但**不能补救 C 的硬伤**。
- **D 的红旗对硬科技成长股权重很高**：核心技术人员流失、激励反复注水、募投长期延期，往往比掌舵人光鲜更能预示新方向落空——好船长配空船照样开不动。
- **第二曲线邻接性直接决定 B 型期权按多大比例计入**（§4 (2)）：内生邻接可观计入，外延跨界默认折价。
- founder 已淡出 / 接班未验证：不直接判"低"，但把"高"封顶到"中性"，并在置信度里点明。
- 别被明星创始人 / 网红老板的人设带偏——以承诺兑现、核心技术人员稳定性等**硬记录**为准。

## 4. 接回定量（全量概率机制）

档位不是写完就算——它要量化接回 `quantitative_framework.md §5.4 / §5.6`。下面是**启发式起点**，模型可按个案微调，但每次调整都要给依据，并把概率显式标注为"主观假设"。

**(1) → §5.4 三情景概率权重**

档位决定三情景（乐观 / 基准 / 压力）的主观概率质量分布，进而算**期望市值 = Σ 概率 × 情景市值**：

| 档位 | 乐观 / 基准 / 压力（起点示例） | 直觉 |
|---|---|---|
| 高成功率 | 40 / 45 / 15 | 概率质量推向乐观，相信团队能把上限兑现 |
| 中性 | 25 / 50 / 25 | 围绕基准、对称分布 |
| 低成功率 | 15 / 40 / 45 | 概率质量推向压力，新方向当作期权而非基准 |

**(2) → B 型期权计入比例**

档位决定 B 型「未被定价的期权」进入主估值路径的比例：

- **高** → 可观计入（期权可作为乐观情景的核心驱动、甚至部分进基准）。
- **中** → 部分计入（仅放乐观情景，基准里不算）。
- **低** → 不进主路径，仅在表后做"若成功则……"的对照脚注。

**(3) → §5.6 整体置信度**

成长成功率档位作为整体置信度的显式输入之一（与六项必输出的完备度、数据时效、财务异常项并列）：

- 档位"低成功率 / 存疑" → 整体结论置信度降一档，并在估值结论里写明"溢价依赖团队兑现、而团队记录存疑"。
- 成长股**完全没做**这一层评估 → 同样视为置信度缺口，降一档。

## 5. 纪律与边界

- **事实 vs 推断分离**：写报告时区分"原文 / 数据看到的"（如某次激励考核未达标、某笔减持）和"据此推断的"（如管理层倾向讲故事）。引用承诺兑现、监管函等结论必须能落到公告 / 财报出处。
- **可推翻假设**：档位是假设不是定论，标注置信度（高 / 中 / 低）和依据；新信息出现可翻案。
- **数据缺口**：骨架模式拿不到承诺兑现 / 监管历史 / 创始人履历是常态——明说"未取得，需 Deep / 联网补全"，对应降档置信度，**不要用财务数据硬凑人 / 治理判断，更不要编造创始人背景**。
- **不踩内幕红线**：只用公开披露（公告、定期报告、合规媒体报道）。不揣测、不暗示未公开信息。
- **不是买卖信号**：高成功率 ≠ 买入、低成功率 ≠ 卖出。它只调节"成长溢价该给多少、置信度多高"，最终结论仍由估值 + 主线归属 + 风险综合得出。
- **避免人设滤镜**：明星创始人 / 网红老板不等于高成功率——以承诺兑现度等硬记录为准，警惕被叙事和个人魅力带偏。
