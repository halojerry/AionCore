# 行业估值方法库（按行业 / 商业模式选估值法）

> 本文件按需加载。`quantitative_framework.md` §5.1 在做"行业 / 商业模式分型"时引用。

定量分析里最常见的系统性错误，不是算错数，而是**用错估值法**——拿通用 PE 去框一只利润尚未释放的高增长公司、或拿 PE band 去看一只重资产 + 强周期的重资产标的、或拿 PEG 去给一只 SaaS 讲故事。

`quantitative_framework.md` §5.1 已给出**生命周期分型**（成长股 / 成熟龙头 / 红利价值）——它回答"看远期还是看当期"。本文件给出第二条正交轴：**行业 / 商业模式分型**——它回答"该用哪个倍数 / 哪个指标作为主估值锚"。**两条轴一起用**：先确定生命周期，再按行业挑指标。例：IC 设计 + 成长股 = PEG / PS 为主 + 远期 PE 兜底；晶圆制造 + 成熟龙头 = PB / EV-EBITDA 为主 + PE band 辅；银行 + 红利价值 = PB / 股息率 / P-EV 三轴并用。

## 1. 选法决策树（先查 §2 行业表，表外才走四问）

**纪律：行业表优先。** 公司所在赛道命中 §2 行业表时，**先按表内"首选估值方法"作为主锚的起点**，决策树只用来检查"是否符合首选方法的前提条件"（如盈利稳定、利润率成熟态）。如果不符合前提，按下面四问调整。

**只有 §2 表里找不到该行业 / 商业模式时**，才直接走四问从零选锚。

回答顺序固定，前一个判断改写后一个判断的权重。

**Q1：盈利是否稳定且可预测，且利润率已接近成熟态？**
- 两个条件**同时**成立 → PE / PEG 可作为主估值法之一（仍要受 Q3 资产结构修正）。
- "稳定但利润率未释放"（如半导体设备、SaaS 早期、机器人）→ **不算 PE 适用**，直接跳到 Q2。判断"利润率未释放"的信号：扣非净利率显著低于同业成熟态、毛利率好但费用率吞噬利润、扣非亏损 / 微利但收入高增、政府补助 / 非经常项目占利润比 > 30%。
- 亏损 / 微利 / 利润大幅波动 → 跳过 PE，进入 Q2。

**Q2：收入是否高增但利润尚未释放？**
- 是 → 主用 **PS / EV-Sales**；增长质量好 + 行业空间清晰时可叠加远期 PE（"达产 / 利润率正常化后的 PE"作为辅助锚）。
- 否，跳过。

**Q3：资产结构是否重 / 是否强周期？**
- 是（折旧高、capex 占收入比高、产能利用率显著影响利润、价格随周期大幅波动）→ 主用 **PB / EV-EBITDA / 中周期利润 PE**，**不能用景气顶利润算 PE**。
- 否，跳过。

**Q4：是否多业务并行、且各业务估值逻辑差异显著？**
- 是 → **SOTP 分部估值**，对每段业务套合适的方法分别估值后求和。
- 否，跳过。

**特别商业模式补丁：**
- **订阅制 / SaaS / ARR 驱动**：主用 **EV/ARR + PS + Rule of 40**；ARR、净留存率（NDR）、毛利率三者缺一不可。
- **创新药 / 临床阶段**：主用 **rNPV / DCF**；按管线成功率与峰值销售推算。
- **金融 / 银行 / 保险**：主用 **PB + P/EV / P/NBV + ROE-PB 联动**；不要单看 PE，因为利润里有大量未实现损益与拨备调节。
- **地产 / 基建**：主用 **PB + NAV**；表内土地储备和合同负债比当期利润更说明问题。

## 2. 行业估值方法库（首选 / 辅助 / 中枢逻辑）

下表是 issue 中给出的库，作为 Q1–Q4 之上的行业级 cheat sheet。**行业归属来自 `stock_basic.industry` 或定性章节的行业判断**；表内方法不是死规则——遇到行业内特殊商业模式（如某半导体公司转型 IDM），按 Q1–Q4 重新分流。

| 行业/板块 | 首选估值方法 | 辅助估值方法 | 估值中枢判断逻辑 |
|---|---|---|---|
| 半导体设备 | PS、EV/Sales、PEG | PE、调整后 PE、EV/EBITDA | 成长期看收入扩张、订单、国产化率；利润率未充分释放时，单纯 PE 容易高估"贵"。设备商和设计商更适合以 PS 或 EV/Sales 为主、PE 为辅 |
| 半导体材料 | PE、PS | EV/EBITDA、PEG | 材料企业一旦通过客户认证，收入持续性较强。成熟材料看 PE，导入期材料看 PS 和客户验证进度 |
| 晶圆制造 / 代工 | PB、EV/EBITDA | PE、EV/产能 | 重资产、高折旧、产能周期明显，利润受折旧和产能利用率影响较大，不宜只看 PE。科创板估值方法中也将 IC 制造更偏向 PB |
| IC 设计 | PE、PEG、PS | PCF、研发费用还原 PE | 轻资产、高研发、周期波动强。高景气阶段不能只看当期 PE，要结合产品周期、库存周期和远期利润 |
| 封测 | PE、EV/EBITDA | PB | 相对制造环节资产重度低一些，但仍受资本开支、稼动率和周期影响 |
| 软件 / SaaS / AI 应用 | PS、EV/Sales、EV/ARR | Rule of 40、FCF、PE | 早期看收入增速、续费率、毛利率、客户留存；成熟后切回 PE / FCF。SaaS 估值通常围绕收入倍数展开，2025 年公开市场 EV/ARR 中位数约在 6x 附近，典型区间约 3x–10x，取决于增长、留存和毛利率 |
| 工业软件 / 信创软件 | PS、PE | EV/Sales、ARR 倍数 | 前期收入增长和客户渗透更重要，后期看利润释放。国产替代、客户粘性、续费率决定估值溢价 |
| 互联网平台 | PE、FCF | PS、SOTP | 成熟平台看利润和自由现金流；多业务平台（电商、广告、云、AI 应用）应做分部估值 |
| AI 算力 / 数据中心 | EV/EBITDA、DCF | PE、P/算力规模、SOTP | 重点看资本开支、上架率、电力成本、合同期限、EBITDA margin |
| 机器人 / 具身智能 | PS、EV/Sales、SOTP | 远期收入情景估值 | 当前利润通常不能反映产业空间。核心看订单验证、量产节奏、BOM 降本、客户结构 |
| 新能源设备 | PE、PEG | EV/EBITDA、订单收入比 | 设备公司适合看订单、收入确认周期、毛利率和新技术路线。高增长阶段用 PEG 修正 PE |
| 光模块 / CPO / 通信设备 | PE、PEG、PS | EV/EBITDA | 高景气周期看订单、客户认证、产品代际升级；但要警惕估值中枢随速率周期切换而波动 |
| 消费品 | PE、DCF | PEG、EV/EBITDA | 成熟消费看品牌力、渠道、现金流和分红；成长消费看渗透率和同店增长 |
| 医药 / 创新药 | rNPV、DCF | PS、PE | 创新药不能简单用 PE。临床阶段看管线成功率和峰值销售，商业化后看 PS / PE |
| CXO / 医疗服务 | PE、PEG | EV/EBITDA | 看订单、产能利用率、客户结构和海外需求周期 |
| 周期资源品（煤炭、有色、化工等） | PB、EV/EBITDA | 中周期 PE、吨盈利 | **不应用景气顶点利润给高 PE**，要用中周期盈利或资产重估 |
| 金融 / 银行 / 保险 | PB、P/EV、P/NBV | PE、ROE-PB 模型 | 银行核心看 PB、ROE、不良率；保险看内含价值和新业务价值 |
| 地产 / 基建 | PB、NAV | PE、EV/EBITDA | 资产负债表质量比当期利润更重要 |
| 军工 / 低空经济 | PE、PS | 订单 / 市值、SOTP | 军工利润释放慢，订单和型号放量往往领先利润；低空经济早期适合情景估值 |

**行业表外的公司怎么办**：先按 Q1–Q4 决策树定主指标；再到表内找**商业模式最相似**的行业作锚；如果还是不确定，结论里写"无标准估值法，主估值锚为 X，置信度低"，比强套 PE 安全。

## 3. 估值中枢的四个来源（动态、不写死数值）

估值中枢**不允许用固定数字**（"半导体设备 PE 50× 算贵"这种话不能写）。中枢由以下四个来源构成，按可得性引用，并显式标注每个来源的时间窗口与样本：

| 中枢来源 | 数据获取 | 优先级 | 注意事项 |
|---|---|---|---|
| **公司自身历史估值区间** | `fetch valuation-band <code> --years 5`，含 PE / PB / PS / 股息率近 1 / 3 / 5 年 min / p25 / median / p75 / max 与当前分位 | **始终先取**——脚本已固化 | 上市时间短 / 高增长样本不足 / 增速降档导致历史中枢失效时，必须缩短窗口或明确弃用 |
| **可比公司当前中位数** | 选 3–5 家同业可比公司（同细分赛道、商业模式、规模量级），对每家 `fetch daily-basic --limit 1`，取首选指标（PE / PB / PS / 股息率等）的中位数 | 几乎所有研报都应给出 | 可比标的的选择本身要写明依据（赛道相同、客户重合、产品代际接近等）；只有 1–2 家可比时不算中位数 |
| **行业历史分位** | 多数 case 需要联网检索（申万 / 中证行业指数估值），脚本无现成数据；不要拍脑袋 | 重要但可降级 | 对行业内的"行业级估值中枢"做引用；若无法获得，明确说"行业中枢未取得，仅以同业中位 + 公司自身历史替代" |
| **产业景气位置** | 联网检索（产业链价格、稼动率、订单等）+ 公司公告 | 强周期 / 政策驱动行业必看 | 用一句话定位"当前景气处于上行 / 见顶 / 下行 / 触底回升"，并说明对中枢的影响方向 |

**写作要求**：
- 至少给出"公司自身历史区间"+"同业可比中位数"两项，否则估值结论置信度直接标"低"。
- 引用同业中位数时，必须列出可比公司名 + 取数日期 + 各家的指标值。
- 行业历史分位与景气位置如果无法取得，**写"未取得"而不是省略**，让用户知道这块缺什么。

## 4. 同业可比公司选取的工作流

脚本目前**没有**行业内成员表 / 自动同业接口（`stock_basic` 只给单字段 `industry`）。所以同业列表必须由你主动构造，工作流如下：

1. **从主营业务拆分定位赛道**：用 `pack` 产出的 `business_structure.product_analysis` 或定性章节判断公司的**核心业务赛道**（不只看 `industry` 字段，往往太粗）。例：兆易创新 `industry`=半导体，但核心赛道是"存储 + MCU"，应选北京君正、聚辰股份、东软载波这类同业，而不是中芯国际。
2. **构造 3–5 家可比清单**：可比标准——同赛道、商业模式相近、规模量级相近（市值差距 < 5x）、客户/产品代际接近。优先选 A 股可比；A 股可比不足时再加 H 股 / 美股（数据接口可能不可得，需联网）。
3. **逐家取最新估值快照**：对每家 `python scripts/data_fetcher.py fetch daily-basic <code> --limit 1`，提取首选指标。
4. **算中位数**：对 3–5 家的首选指标取中位（不取均值——避免极端值拉偏）。
5. **写进报告**：在"估值中枢"段列表展示可比公司 + 各家指标 + 中位数 + 取数日期。

可比公司列表**本身是一个可被推翻的假设**——必须说明选择依据，并允许"如果用户认为某家不可比，可以替换重算"。

## 5. 上修 / 下修条件的写作模板

估值结论不写"贵 / 合理 / 便宜"就收口。**必须显式给出会让估值上修和下修的具体事件 / 数据信号**，让读者知道该跟踪什么。

**上修条件**（什么发生会让当前估值看起来更合理 / 甚至偏低）：
- 业绩 / 财务类：营收增速连续 N 期超 X%、毛利率回到 X%、ROE 重新站上 X%、FCF 转正、扣非利润同比由负转正等。
- 行业 / 产业类：行业景气从下行转上行、主线轮动至本赛道、关键产品价格回升至 X、稼动率回到 X%。
- 公司事件类：拿到核心客户大单、新产品验证通过、产能瓶颈解决、出海突破。
- 估值 / 流动性类：可比公司估值中枢上移、行业指数估值修复、北向 / 公募显著增持（仅作辅助）。

**下修条件**（什么发生会让当前估值显得偏高 / 该警惕）：
- 业绩 / 财务类：增速降档（连续 N 期低于 X%）、毛利率压缩 N 个百分点、应收 / 存货增速持续超收入、商誉减值、OCF / 净利长期 < 0.5。
- 行业 / 产业类：景气见顶回落、主线退潮、价格战、产能过剩、政策收紧。
- 公司事件类：核心客户流失、关键产品验证失败、监管 / 问询信号、管理层 / 大股东减持。
- 估值 / 流动性类：可比公司估值中枢下移、北向 / 公募显著减持、解禁 / 大宗交易压力集中。

**模板示例**：

```
合理估值区间：[基于同业可比中位 + 公司自身 3 年 median 给出区间，注明窗口]
当前估值：[当前 PE/PB/PS/股息率，注明日期]
估值阶段：[相对区间处于下沿/中枢/上沿/突破]，置信度：[高/中/低]

上修条件（任一成立则估值锚可上修）：
- [条件 1，含量化阈值或事件信号]
- [条件 2 …]

下修条件（任一成立则估值锚需下修）：
- [条件 1，含量化阈值或事件信号]
- [条件 2 …]
```

## 6. 跟历史框架的兼容（生命周期 × 行业 双轴）

`quantitative_framework.md` §5.1 的**生命周期分型**与本文件的**行业 / 商业模式分型**互相正交，不冲突：

| 双轴示例 | 生命周期 | 行业方法（本文件） | 主估值锚 |
|---|---|---|---|
| 处于扩产期的半导体设备龙头 | 成长股 | PS、EV/Sales、PEG | **PS 历史分位 + 同业 PS 中位**；远期 PE 兜底；PE band 仅作下行参考 |
| 增速降档后的消费品龙头 | 成熟龙头 | PE、DCF | **PE 历史分位 + 股东回报率**；同业 PE 中位辅助 |
| 高股息煤炭 / 银行 | 红利价值 | PB、股息率、P/EV | **股息率分位 + PB 分位**；与 10 年国债利差核对 |
| 利润尚未释放的创新药 | 成长股 | rNPV、DCF | **rNPV 情景估值**；PE / PS 失效 |
| 平台型互联网 | 成熟龙头（部分业务）+ 成长股（新业务） | SOTP | **分部估值**，每段套对应方法 |

**判断顺序**：
1. 先做生命周期分型（§5.1 三型）。
2. 再做行业 / 商业模式分型（本文件 §1–§2）。
3. 两轴决定主估值锚 + 辅助锚。
4. 按本文件 §3 取中枢、§5 写上修 / 下修条件。
